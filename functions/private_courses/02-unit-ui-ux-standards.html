視覺階層與認知負荷

⚠️

色彩編碼

顏色傳遞訊息，不需文字。紅色與黃色能瞬間抓住注意力，綠色則給人安全感。

📏

大小差異

最重要的功能應該最大。根據 Fitts's Law，目標越大、距離越近，操作所需時間越短。

Primary
🧠

資訊降噪

FPV 競速時，駕駛者沒有時間閱讀。只顯示當下必要的數據，隱藏次要資訊，降低認知負荷。

🎯 學習目標

預習如何減少操作者的認知負擔，讓使用者在高速飛行時不需要「思考」按鈕在哪裡，而是憑「直覺」操作。

UI/UX 設計規範：從視覺到體驗

【作業 3-1】「盲操」佈局挑戰 (可用性測試)

  • GitHub 角色： 可用性分析師。
  • 執行方式： 根據 Thumb Zone 理論重新排列「停止」與「前進」按鈕。
  • 評分重點： 按鈕大小（觸控熱區）是否足夠？是否避開了螢幕頂端的難觸及區？

【作業 3-2】FPV HUD 抬頭顯示器 (視覺層級)

  • GitHub 角色： 視覺設計師。
  • 執行方式： 將遙測數據（Ping/電量）以半透明方式疊加在影像容器上。
  • 評分重點： 視覺層級的處理，以及 CSS 透明度與圖層順序（z-index）的應用。

【作業 3-3】異常狀態 UX 處理 (回饋設計)

  • GitHub 角色： 使用者體驗官。
  • 執行方式： 設計「斷線」或「弱訊號」時的視覺提醒（如紅色閃爍）。
  • 評分重點： 顏色編碼（Color Coding）的直覺性，以及過渡動畫對注意力的引導。

4. 翻轉教室中的評分標準 (Checkpoint)

任務 評分標準 (透過 GitHub 檢查) 課堂討論議題
3-1 核心按鈕是否位於「自然操作區（Natural Zone）」？ 如果使用者是左撇子，目前的 UI 該如何調整？
3-2 HUD 資訊是否精簡？有無遮擋中央 50% 的核心視野？ 在強烈陽光下，哪種顏色的 HUD 文字辨識度最高？
3-3 回饋機制是否即時？是否正確使用顏色傳遞緊張感？ 除了視覺，還有哪些方式可以提醒使用者「斷線了」？

總結評價

單元 2 與 3 的作業旨在建立學生從代碼到體驗的轉化能力。在 GitHub Classroom 上，老師可以透過學生提交的 UI 截圖或代碼結構，快速判斷其「排版邏輯」與「設計直覺」。建議課堂上讓同學交換手機測試對方的 UI，實機體驗所謂的「拇指操作區」。
