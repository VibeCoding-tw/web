基礎 9 - 單元 2

精通 硬體定時器
& ISR 中斷服務

超越 millis() 的極致精確度，讓處理器為你「插隊」執行關鍵任務。

⏰ 1. 什麼是硬體定時器？ (30 min)

核心概念：想像處理器內部有一個獨立的小鬧鐘。主程式（Loop）在跑的時候，鬧鐘時間一到，處理器會「強行暫停」手邊工作，去執行特定任務。

⚔️ 與 millis() 的差異

millis()：被動查詢

「看錶」，必須在 loop 裡面手動檢查時間，會受其他程式阻塞影響。

定時器中斷：主動通知

「鬧鐘響」，時間到直接插隊執行，不受 loop 阻塞影響，精度極高。

🎯 學習目標

理解 ESP32 擁有 4 個硬體定時器，可以達到微秒 (\(\mu s\)) 等級的精確度。

🚫 2. 中斷服務常式 (ISR) 的三大禁忌 (30 min)

關鍵概念：ISR 必須「快進快出」。

不可使用 delay()

會導致系統鎖死，因為中斷已經停止了時間流逝的計數環境。

不可進行耗時運算

例如複雜的 JSON 解析或大容量浮點運算。這會卡住主程式太久。

不可使用 Serial.print()

I/O 操作太慢，會干擾中斷時序，甚至導致 Watchdog 重新啟動。

⚡ 加速關鍵：IRAM_ATTR

使用 IRAM_ATTR 關鍵字，可以確保 ISR 程式碼存放在 RAM 中，而不是從較慢的 Flash 讀取，確保中斷反應即時。

void IRAM_ATTR onTimer() {
  // 內容越精簡越好
}

🛡️ 3. 變數共享：volatile 關鍵字 (20 min)

核心概念：當主程式與 ISR 都會用到同一個變數（如：計數器）時，必須加上 volatile。

這會防止編譯器為了優化速度，而把變數存在暫存器中「懶得讀取」，導致主程式讀到舊的數值。

核心任務：硬體級別的精確控制

運用硬體中斷，實現那些不容許延遲的關鍵功能。

🚨 60 min

【作業 2-1】

「硬體定時器閃燈」

利用硬體定時器，每隔精確的 \(1,000,000 \mu s\) (1秒) 切換 LED 狀態。

挑戰：在 loop 裡寫 delay(5000)，觀察 LED 是否依然能精確閃爍？
🏎️ 60 min

【作業 2-2】

「轉速計實作」

利用定時器每 100ms 觸發中斷，計算這段時間內馬達轉了幾圈。

技術：學習使用 portENTER_CRITICAL 來保護共享數據。
🛡️ 45 min

【作業 2-3】

「硬體級 Watchdog」

實作一個「看門狗」，如果藍牙 2 秒內沒收到心跳指令，定時器自動強制停止馬達。模擬當 loop 陷入死迴圈時，定時器如何發揮保護作用。
