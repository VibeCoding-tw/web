基礎 6 - 網路通訊基礎

數位世界的溝通橋樑：
HTTP 協議與 Fetch API

從餐廳點餐模型出發，掌握讓網頁、伺服器與無人車互相「對話」的核心技術。

🍽️ 1. 餐廳模型：理解 HTTP 對話 (20 min)

HTTP (HyperText Transfer Protocol) 是網路上交換數據的標準格式。最簡單的理解方式就是「餐廳點餐」：

🙋‍♂️ 請求 (Request)

你向服務生點餐。包含你想看的內容（網址）、你想做的動作、以及你提供的資訊。

🍛 回應 (Response)

服務生送來餐點。包含狀態（成功或賣完）以及具體的內容（網頁、圖檔或 JSON）。

🧠 無狀態性 (Stateless)

伺服器很健忘！每次連線都是獨立的，它不會主動記得你是誰（除非使用 Cookie 或 Token）。

💡 概念解析

為什麼「無狀態性」很重要？這讓伺服器可以同時服務成千上萬的人，而不需要為每個人保留一個龐大的記憶庫。

✉️ 2. 請求的組成：方法、標頭與正文 (40 min)

一封完整的 HTTP 請求信件包含以下關鍵部分：

  • GET

    像看菜單，僅請求數據。

    例如：讀取當下無人車的感測器數值。

  • POST

    像填訂單，發送新數據。

    例如：將無人車路徑紀錄傳回伺服器進行分析。

  • Headers

    信封上的備註。

    告訴伺服器我傳的是 JSON 格式，或者我用的瀏覽器版本。

// URL 參數 (Query Strings) 範例
http://192.168.4.1/move?dir=forward&speed=200

🏷️ 3. 狀態碼：伺服器的表情符號 (20 min)

當伺服器收到請求後，會回傳一個三位數的「狀態碼」來告訴你結果：

✅ 200 OK

請求成功！資料已經準備好送給你了。

🔍 404 Not Found

找不到資源。可能是網址打錯了，或者檔案被刪除了。

💥 500 Error

伺服器內部錯誤。通常是後端的程式碼寫壞了。

試著在瀏覽器網址列輸入下方網址，並觀察 ESP32 的序列監控視窗接收到的字串：

作業指引 (Assignment Guide)

【作業 1-1】Postman GET 測試與回傳解析

  • 角色扮演： API 測試工程師 (API Test Engineer)。
  • 執行方式： 使用 Postman 或瀏覽器開發者工具，發送 GET 請求至公開 API（如 JSONPlaceholder）。觀察 Headers、Status Code 與 Response Body，並記錄解析後的 JSON 數據。
  • 評分重點 : 成功解析 JSON 結構，並能準確說明 200 OK 與 404 Not Found 的觸發場景。

【作業 1-2】實作 POST 發送 JSON 數據

  • 角色扮演： 後端數據整合師 (Backend Data Integrator)。
  • 執行方式： 實作一個 POST 請求，將模擬的感測器數據（如 { "temp": 25, "hum": 60 }）發送至測試端點。確保 Content-Type 設定為 application/json。
  • 評分重點 : 數據成功送達伺服器並被正確識別，Payload 格式遵循標準 JSON 規範。

【作業 1-3】HTTP 狀態碼攔截與錯誤處理

  • 角色扮演： 韌體穩定性工程師 (Firmware Stability Engineer)。
  • 執行方式： 實作錯誤處理邏輯。當請求發生逾時或收到 500 錯誤時，ESP32 必須能攔截該狀態並顯示自定義的錯誤訊息，而非讓系統直接當機。
  • 評分重點 : 具備完善的狀態碼判斷邏輯，能有效提升系統的通訊強韌度。

B. 參考範例代碼 (Reference Example Code)

main.cpp (HTTP POST JSON)

#include <HTTPClient.h>

void sendData() {
  HTTPClient http;
  http.begin("http://your-server.com/api");
  http.addHeader("Content-Type", "application/json");
  
  String payload = "{\"status\": \"active\"}";
  int httpResponseCode = http.POST(payload);
  
  if (httpResponseCode > 0) {
    String response = http.getString();
    Serial.println(httpResponseCode);
  }
  http.end();
}

C. 自動評分 (Autograding) 的設計

  • 函式調用檢查：： 驗證是否正確使用 http.addHeader() 設定 JSON 格式。
  • 錯誤攔截驗證：： 透過測試代碼檢查是否包含 if (httpResponseCode > 0) 的防錯結構。
