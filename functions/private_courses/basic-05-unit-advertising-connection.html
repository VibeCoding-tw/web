我是誰？我在哪？
BLE 廣播與連線機制全解析

理解 BLE 如何在擁擠的 2.4GHz 頻段中「聰明生存」，並建立穩定的連線。

📢 1. 廣播機制：我是誰？我在哪？ (20 min)

在連線建立之前，設備必須主動宣告自己的存在。這就是廣播 (Advertising)。

📡 避開干擾的頻道

BLE 廣播固定在 頻道 37, 38, 39。這些頻道的選擇是為了完美避開 Wi-Fi 常用的頻道 1, 6, 11，確保廣播能被順利接收。

📦 封包內容

包含設備名稱、MAC 地址、發射功率 (TX Power) 與 Service UUID。這讓手機知道「我是一台遙控車」。

⏱️ 廣播間隔 (Advertising Interval)

  • 靈敏模式 (20ms): 極快被搜尋到，但非常耗電。
  • 省電模式 (1000ms): 非常省電，但可能需要掃描好幾秒才看得到。

💡 學習任務

觀察你的手機藍牙列表，哪些設備是「可以連線的」，哪些只是在「廣播數據」（如百貨公司的 iBeacon 廣告）？

🤝 2. 掃描與連線流程 (25 min)

當手機發現你的 ESP32 後，會經歷以下階段建立專屬通道：

  1. 1
    掃描 (Scanning)

    手機 (Central) 在廣播頻道間切換，傾聽空中的封包。

  2. 2
    連線請求 (Connection Request)

    手機決定連線，發送時序指令，約定：「之後我們在特定頻率跳頻溝通，別再廣播了」。

  3. 3
    連線成功

    ESP32 停止廣播，進入 1 對 1 的高速數據交換模式。這就是為什麼 A 同學連線後，B 同學就找不到了。

🔁

頻率跳頻 (AFH)

連線後，設備會在剩餘的 37 個頻道中快速跳轉，這讓 BLE 即使在 Wi-Fi 干擾嚴重的環境下依然穩定。

🔋 3. 省電的關鍵：工作週期 (15 min)

無人車如果一直維持高頻率廣播，電池很快就會耗盡。優秀的設計會考慮工作週期 (Duty Cycle)：

⏱️ 限制時間廣播

例如：開機後僅廣播 30 秒，若無人連線則進入休眠。下次需按 Reset 鍵喚醒。

🔘 按鈕觸發廣播

只有在按下配對鈕時才啟動廣播，平常完全關閉藍牙功能以達到極致省電。

"BLE 的低功耗，不是因為它傳輸快，而是因為它大部份時間都在睡覺。"

作業指引 (Assignment Guide)

【作業 2-1】「改名大師」：自定義廣播封包

  • 角色扮演： 品牌設備工程師 (Brand Device Engineer)。
  • 執行方式： 修改設備名稱（包含個人座號，如 Car_07）。使用手機 nRF Connect App 掃描並確認名稱是否正確顯示，進階挑戰：嘗試將 MAC 地址最後三碼加入名稱中。
  • 評分重點 : 成功修改廣播名稱且格式正確，能在手機端被唯一辨識。

【作業 2-2】「連線守門員」：連線回呼 (Callback)

  • 角色扮演： 通訊系統開發員 (Communication System Developer)。
  • 執行方式： 實作 BLEServerCallbacks。當手機連線時亮起綠燈並在 Serial 印出連線成功訊息；斷線時則亮起紅燈，並務必自動呼叫 startAdvertising() 重新啟動廣播。
  • 評分重點 : 回呼函式運作正常，且斷線後能自動回復廣播狀態，確保下次能再次被搜尋連線。

【作業 2-3】「距離與強度的秘密」：RSSI 測距

  • 角色扮演： 場域感知工程師 (Field Awareness Engineer)。
  • 執行方式： 保持手機與 ESP32 的不同距離（30cm, 1m, 3m），記錄 RSSI 強度的變化。實作簡單邏輯：若 RSSI > -50dBm (極近距離) 則讓板載 LED 閃爍提示。
  • 評分重點 : 撰寫完整的 RSSI 記錄表，並能描述訊號強度與物理距離的非線性關係。

B. 參考範例代碼 (Reference Example Code)

main.cpp (BLE 重連回呼)

#include <BLEDevice.h>
#include <BLEServer.h>

class MyServerCallbacks: public BLEServerCallbacks {
    void onConnect(BLEServer* pServer) {
      Serial.println("Device connected!");
    };
    void onDisconnect(BLEServer* pServer) {
      Serial.println("Device disconnected!");
      BLEDevice::startAdvertising(); // 自動恢復廣播
    }
};

void setup() {
  BLEDevice::init("ESP32_CAR_01");
  BLEServer *pServer = BLEDevice::createServer();
  pServer->setCallbacks(new MyServerCallbacks());
  BLEDevice::startAdvertising();
}

C. 自動評分 (Autograding) 的設計

  • 核心配置掃描：： 檢查 BLEDevice::init() 的調用參數是否符合命名規範。
  • 流程邏輯驗證：： 透過模擬執行確認斷線回呼中是否存在 startAdvertising。
