🎼 1. 數位轉類比的橋樑：LEDC 三部曲 (20 min)

⏱️ 定時器 (Timer)

決定 PWM 的「心跳」，定義頻率與解析度的上限。

📡 頻率 (Frequency)

每秒切換幾次？馬達通常建議使用 1kHz ~ 20kHz。

🧩 解析度 (Resolution)

精細度。8-bit 為 0~255；10-bit 為 0~1023。

🔢 頻道 (Channel)

ESP32 有 16 個獨立頻道 (0~15)，可綁定不同腳位。

🔗 綁定 (Attachment)： 將設定好的「頻道」指向具體的「GPIO 腳位」。

💡 硬體架構觀念

  • ● 多個頻道可以共用同一個定時器。
  • ● 共用定時器的頻道，其頻率與解析度必須相同。
  • ● 占空比 (Duty) 則是每個頻道可以獨立設定的。

⚙️ 2. 關鍵語法與 API 流程 (20 min)

ledcSetup(channel, freq, res);

設定頻道的參數：哪個頻道、多快、多精細？

ledcAttachPin(pin, channel);

把物理腳位接上頻道：哪個 GPIO 要傳出訊號？

ledcWrite(channel, duty);

輸出占空比：讓它轉多快或亮多亮？

🔍 學習任務

查閱資料，如果我有兩個馬達需要不同的頻率（例如一個控制泵，一個控制移動馬達），我該如何分配 Timer 與 Channel？

CODE SNIPPET
// 初始化設定
ledcSetup(0, 5000, 8);  // CH:0, 5kHz, 8-bit
ledcAttachPin(2, 0);    // GPIO 2 -> CH:0

// 循環控制
for(int i=0; i<255; i++) {
  ledcWrite(0, i);      // 漸亮
  delay(10);
}

❓ 3. 為什麼不直接用 analogWrite？ (15 min)

精確控制： LEDC 允許你在程式執行中動態修改頻率，這對音效產生（如蜂鳴器）或伺服馬達（Servo）極其重要。

資源優化： 你可以決定哪些腳位共用同一組時脈源，在複雜的低功耗或高性能任務中具有更高的靈活性。

作業指引 (Assignment Guide)

【作業 3-1】「頻道接力賽」：多路 PWM 輸出

  • 角色扮演： 硬體資源管理員 (Hardware Resource Manager)。
  • 執行方式：： 同時控制一個 LED 與兩個馬達。將 Channel 0 分配給 LED (5kHz)，Channel 1 & 2 分配給馬達 (15kHz)。觀察並記錄修改 CH1 頻率是否會影響 CH2 的行為。
  • 評分重點 : 能正確配置多個 LEDC 通道與定時器，且清楚定時器資源在通道間的共享邏輯。

【作業 3-2】「百分比速度控制器」：API 封裝

  • 角色扮演： 軟體架構實習生 (Software Architecture Intern)。
  • 執行方式：： 撰寫一個封裝函式 setMotorSpeed(int percent)。輸入 0~100 並自動轉換為對應解析度（例如 10-bit 為 1023）的 ledcWrite 數值，讓主程式代碼更易讀。
  • 評分重點 : 成功實作映射 (Mapping) 邏輯，且封裝後的函式在主程式中運作正常，代碼整潔度高。

【作業 3-3】「頻率掃描器」：尋找馬達諧振點

  • 角色扮演： 音響/震動測試專家 (Acoustic/Vibration Specialist)。
  • 執行方式：： 寫一個迴圈讓馬達頻率從 100Hz 漸增到 40kHz。紀錄在哪個頻率下高頻雜音最小？並觀察頻率過高時馬達轉動力的變化。
  • 評分重點 : 撰寫完整的掃描實驗紀錄，包含頻率、雜音程度與轉動力觀察，並給出建議的最佳運作頻率。

B. 參考範例代碼 (Reference Example Code)

main.cpp (LEDC 四大要素配置)

#include <Arduino.h>

const int ledPin = 18;
const int channel = 0;
const int freq = 2000;
const int resolution = 13; // 13-bit (0-8191)

void setup() {
  // 1. 配置頻道參數
  ledcSetup(channel, freq, resolution);
  // 2. 將頻道連結至實體 GPIO
  ledcAttachPin(ledPin, channel);
}

void loop() {
  ledcWrite(channel, 4096); // 50% Duty Cycle
  delay(1000);
}

C. 自動評分 (Autograding) 的設計

  • 參數合法性掃描：： 驗證頻率與解析度是否超限（ESP32 限制：Freq * 2^Res < 80MHz）。
  • 結構完整性檢查：： 偵測 ledcAttachPin 的掛載參數，確保通道分配無衝突。
