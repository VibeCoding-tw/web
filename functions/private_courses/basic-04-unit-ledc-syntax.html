🎼 1. 數位轉類比的橋樑：LEDC 三部曲 (20 min)

⏱️ 定時器 (Timer)

決定 PWM 的「心跳」，定義頻率與解析度的上限。

📡 頻率 (Frequency)

每秒切換幾次？馬達通常建議使用 1kHz ~ 20kHz。

🧩 解析度 (Resolution)

精細度。8-bit 為 0~255；10-bit 為 0~1023。

🔢 頻道 (Channel)

ESP32 有 16 個獨立頻道 (0~15)，可綁定不同腳位。

🔗 綁定 (Attachment)： 將設定好的「頻道」指向具體的「GPIO 腳位」。

💡 硬體架構觀念

  • ● 多個頻道可以共用同一個定時器。
  • ● 共用定時器的頻道，其頻率與解析度必須相同。
  • ● 占空比 (Duty) 則是每個頻道可以獨立設定的。

⚙️ 2. 關鍵語法與 API 流程 (20 min)

ledcSetup(channel, freq, res);

設定頻道的參數：哪個頻道、多快、多精細？

ledcAttachPin(pin, channel);

把物理腳位接上頻道：哪個 GPIO 要傳出訊號？

ledcWrite(channel, duty);

輸出占空比：讓它轉多快或亮多亮？

🔍 學習任務

查閱資料，如果我有兩個馬達需要不同的頻率（例如一個控制泵，一個控制移動馬達），我該如何分配 Timer 與 Channel？

CODE SNIPPET
// 初始化設定
ledcSetup(0, 5000, 8);  // CH:0, 5kHz, 8-bit
ledcAttachPin(2, 0);    // GPIO 2 -> CH:0

// 循環控制
for(int i=0; i<255; i++) {
  ledcWrite(0, i);      // 漸亮
  delay(10);
}

❓ 3. 為什麼不直接用 analogWrite？ (15 min)

精確控制： LEDC 允許你在程式執行中動態修改頻率，這對音效產生（如蜂鳴器）或伺服馬達（Servo）極其重要。

資源優化： 你可以決定哪些腳位共用同一組時脈源，在複雜的低功耗或高性能任務中具有更高的靈活性。

講師手冊 (Instructor Guide)

1. 教學重點 (Teaching Points)

  • 解析 LEDC 的四大要素：Timer (時鐘源), Frequency (頻率), Resolution (解析度), Channel (通道)。
  • 介紹 ESP32 的 16 個獨立通道分配邏輯。
  • 強調 `ledcAttachPin` 與 `ledcWrite` 的正確調用順序。

2. 常見問題 (Common Issues)

  • 頻率與解析度超限：如果設為 5000Hz 卻用 20-bit 解析度，會導致初始化失敗。提醒學生參考技術手冊的限制公式。
  • 忘記 `ledcAttachPin`：語法上 `ledcSetup` 只是設定硬體參數，必須掛載到實體引腳才有輸出。
  • 通道重複使用：確保每個獨立控制的負載都有專屬的 Channel ID。

3. 評分標準 (Grading Criteria)

  • 基礎分：能正確使用 LEDC 函式控制單路 LED。
  • 進階分：能理解 10-bit (0-1023) 與 8-bit (0-255) 在 `ledcWrite` 時的數據對應關係。
