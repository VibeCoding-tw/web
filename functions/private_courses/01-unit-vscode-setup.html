🛠️ 直觀體驗：從零開始的基建

打地基的時間到了！讓我們用 15 分鐘完成最強編輯器的部署。

1
下載與安裝 (15 min)

前往 code.visualstudio.com，根據你的作業系統 (Windows/macOS/Linux) 下載。

Windows 用戶關鍵設定：

安裝時務必勾選 「加入 '開啟以程式碼編輯' 動作到 Windows 檔案總管內容功能表」 (Add "Open with Code")。

💡 這讓你能在資料夾直接按右鍵開啟專案，效率提升 10 倍！

2
繁體中文介面設定 (5 min)

  • 開啟 VS Code。
  • 點擊左側工具欄最下方的四方塊圖示 (Extensions) 或按 Ctrl+Shift+X。
  • 搜尋 Chinese，選擇 "Chinese (Traditional) Language Pack" 安裝。
  • 重啟 VS Code，介面即刻中文化。

課程教材 - 內部使用

作業指引 (Assignment Guide)

作業 2-1：安裝硬體大腦 PlatformIO

  • 角色扮演： 韌體工程師.
  • 執行方式： 在 VS Code 安裝 PlatformIO IDE 擴充功能，並截圖 PIO Home 畫面。
  • 評分重點： 確認 👽 圖示出現且 PIO Core 已成功安裝（以便開發 ESP32）。

作業 2-2：Git 帳號同步

  • 角色扮演： 雲端同步專家。
  • 執行方式： 在 VS Code 登入 GitHub 並開啟 Settings Sync。
  • 評分重點： 左下角 👤 圖示顯示已登入帳號。

GitHub Classroom 詳細操作指南

第一步：準備範本倉庫 (Template Repo)

  1. 上傳包含 extensions.json 的範本倉庫並勾選 "Template repository"。

第二步：建立 Classroom 作業

  1. 建立 Individual Assignment，關聯範本並啟用 "Feedback Pull Requests"。

第三步：學生端操作 (Flipped Workflow)

  1. 學生領取作業 Clone 到本機，VS Code 會提示安裝推薦外掛。
  2. 完成安裝與 GitHub 登入後，完成檢查表並 Push 代碼。

第四步：教師回饋 (Review)

  1. 進入 Feedback PR，檢核學生提供的環境檢測截圖與 NVS/PIO 配置理解度。

在 GitHub Classroom 的實戰部署建議

建議為 VS Code 環境架設建立一個 "Dev Environment" 驗證作業，建議範本倉庫設置如下：

A. 範本倉庫 (Template Repo) 結構

.
├── platformio.ini   # 預置的 ESP32 設定檔，測試擴充功能是否能識別
├── .vscode/         # 包含推薦擴充功能清單 (extensions.json)
└── env-check.md     # 環境檢查表，學生需勾選完成項目

B. 參考範例代碼 (Reference Example Code)

platformio.ini (ESP32 基礎設定)

[env:esp32c3]
platform = espressif32
board = esp32-c3-devkitm-1
framework = arduino
monitor_speed = 115200

.vscode/extensions.json (推薦外掛清單)

{
    "recommendations": [
        "platformio.platformio-ide",
        "ms-ceintl.vscode-language-pack-zh-hant"
    ]
}

env-check.md (環境檢查清單)

# 開發環境檢查表
- [ ] VS Code 已安裝 (含中文包)
- [ ] PlatformIO 👽 圖示已出現
- [ ] GitHub 帳號已登入並開啟 Sync
- [ ] 成功打開此專案並識別 platformio.ini

C. 自動評分 (Autograding) 的設計

  • 設定檔檢查： 檢查 env-check.md 是否已填寫。
  • 環境截圖： 要求學生將 VS Code 視窗（需含 PIO 圖示與登入狀態）截圖存入 Repo，手動點評。

翻轉教室的實施策略

階段 學生行動 (Git Workflow) 教師/平台角色
課前 (預習) 根據講義下載 VS Code，完成繁體中文化。 支援：在討論區排除 Windows/Mac 下載版本不相符的問題。
課中 (實作) 課堂安裝 PlatformIO（大檔案下載），並開啟 Sync 同步。 巡堂：檢查誰卡在「PIO Core 載入中」，協助手動除錯。
課後 (精進) 建立一個空白 PIO 專案，上傳至 GitHub 觀察環境是否一致。 Feedback PR：檢核學生提供的 PIO 設定截圖。
