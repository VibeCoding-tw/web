從 Arduino IDE 畢業，邁向專業開發之路。
IntelliSense、版本控制、多環境管理，一次滿足。

🚀 為什麼要捨棄 Arduino IDE？ (15 min)

PlatformIO 不僅是好用，它是讓你從「寫作業」變成「做專案」的關鍵工具。

✨

代碼補完 (IntelliSense)

就像寫作文有自動校正。打出 `Serial.` 就會跳出所有可用函式，大幅減少拼錯關鍵字 (Typo) 的機率，寫程式不再靠運氣。

📦

版本管理

PIO 可以幫你鎖定特定版本的函式庫。避免發生「今天能跑，明天函式庫更新就壞掉」的悲劇，專案穩定度 UP。

🌍

多環境編譯

同一個專案，換個設定就能同時燒錄到 ESP32、Arduino Uno 或 ESP8266。不需要複製貼上三份程式碼。

🛠️ 安裝與初始化 (25 min)

1

安裝 VS Code (Visual Studio Code)

前往官網下載並安裝適合你作業系統的版本。

2

安裝 PlatformIO IDE 擴充套件

在 VS Code 左側點擊「方塊圖示 (Extensions)」，搜尋 "PlatformIO IDE" 並安裝。
提示 安裝後通常需要重啟 VS Code，看到左側出現「外星人圖標 👽」即代表成功。

3

認識核心：platformio.ini

這是專案的「大腦」。它紀錄了板子型號 (board)、框架 (framework) 與通訊速率。

[env:esp32dev] platform = espressif32 board = esp32dev framework = arduino monitor_speed = 115200

📂 專案結構巡禮 (20 min)

📄
src/main.cpp
程式碼的主戰場。記得手動加上 #include <Arduino.h>。
📚
lib/
存放你自己寫的私有函式庫 (Private Libraries)。
🗑️
.pio/
編譯產生的暫存檔。不需要去動它，也不要上傳到雲端。
⚙️
platformio.ini
專案設定檔 (Project Configuration File)。
🏗️

PlatformIO 的結構雖然比 Arduino IDE 複雜，
但這正是它能管理大型專案的原因。
習慣後，你會回不去的！

課程教材 - 內部使用

作業指引 (Assignment Guide)

1. GitHub Classroom 設定

  • 點擊儀表板「領取作業」按鈕，獲取 `basic-01-pio-setup` 作業儲存庫。
  • 將專案 Clone 至本地，確保專案路徑不含中文或特殊字元。
  • 使用 VS Code 開啟資料夾，等待 PlatformIO 完成環境初始化。

2. 翻轉課堂學習策略

  • 課前預習：觀看「PlatformIO 安裝與基礎操作」教學影片。
  • 課中實作：完成第一個 HelloWorld 專案並嘗試修改 `platformio.ini`。
  • 課後思考：相較於 Arduino IDE，PlatformIO 的專案結構（如 src, lib）有什麼優點？

3. 單元實作任務

  • 作業 1-1 (40%)：成功建立專案並通過 Build（編譯）流程，截圖編譯成功畫面。
  • 作業 1-2 (30%)：修改代碼使 Serial 每隔 500ms 印出「Vibe Coding Ready」。
  • 作業 1-3 (30%)：在 `platformio.ini` 中新增一個指定的 Library（如 ArduinoJson）並編譯通過。

GitHub Classroom 詳細操作指南

第一步：準備範本倉庫 (Template Repo)

  1. 上傳包含基礎 Hello World 但「故意缺少必要函式庫設定 (lib_deps)」的專案框架，並勾選 "Template repository"。

第二步：建立 Classroom 作業

  1. 建立作業並連結範本。建議開啟 Feedback Pull Requests 選項以便對比學生的 IDE 配置與依賴管理邏輯。

第三步：學生端操作 (Flipped Workflow)

  1. 學生領取作業，獲得個人專屬 Repo，設定編譯環境、安裝依賴庫、達成編譯成功並印出驗證訊息。
  2. 本地開發後 Push 代碼。

第四步：教師回饋 (Review)

  1. 打開 Feedback PR，針對學生的「IntelliSense 障礙排除思維」與「編譯日誌 (Build Log) 分析能力」予以具體回饋。

在 GitHub Classroom 的實戰部署建議

此單元引導學生從傳統 IDE 轉向現代化的 CLI+Ext 開發流，掌握工程化的專案依賴管理工具，建議範本倉庫設置如下：

A. 範本倉庫 (Template Repo) 結構

.
├── platformio.ini   # (重點) 需包含 upload_speed 與 lib_deps 練習
├── src/main.cpp      # 包含 Serial "Ready" 輸出邏輯
└── README.md        # 編譯成功截圖、專案目錄層次分析與 Library 安裝紀錄

B. 參考範例代碼 (Reference Example Code)

platformio.ini (依賴管理練習)

[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
monitor_speed = 115200
upload_speed = 921600

; 學生作業：需自行新增 ArduinoJson 依賴
lib_deps =
    bblanchon/ArduinoJson @ ^6.21.0

C. 自動評分 (Autograding) 的設計

  • 依賴項檢核： 使用 Actions 解析 platformio.ini，確認 lib_deps 中是否正確包含了指定版本。
  • 結構驗證： 檢測是否包含 .gitignore 以確保學生理解「不提交編譯產物」的工程規範。

翻轉教室的實施策略

階段 學生行動 (Git Workflow) 教師/平台角色
課前 (預習) 觀看「為什麼我們選擇 VS Code + PlatformIO 而非 Arduino IDE？」影片並預先安裝 PIO 插件。 Dashboard：確認學生已完成 IDE 初期配置，準備好跨平台編譯環境。
課中 (實作) 領取作業對比不同 OS 下初始化專案的時間差與常見 C++ 語法高亮 (IntelliSense) 紅線問題。 助教角色：引導學生理解 Rebuild IntelliSense Index 的必要性與 #include <Arduino.h> 在 PIO 中的地位。
課後 (精進) 實作具備 Library 呼叫邏輯的範本專案並 Push 最终 README 分析报告。 Feedback PR：審核學生對專案目錄結構理解度與依賴版本控制的嚴謹度。
