基礎 6 - 非同步編程

從等待到非同步：
JavaScript Fetch API 基礎

學習如何發送一個「背景請求」，讓網頁在不重新整理的情況下，優雅地與遠端伺服器交換數據。

⏳ 1. 從 XHR 到 Fetch (20 min)

在過去，我們使用 XMLHttpRequest (XHR) 來處理 AJAX，但它的語法冗長且基於事件監聽，容易導致「回呼地獄 (Callback Hell)」。

為什麼選擇 Fetch？

  • 基於 Promise：結構清晰，易於鏈結。
  • 語義化：API 命名直覺（fetch = 去拿）。
  • 簡潔：減少了大量的模板代碼。

Promise 回顧

PENDING 請求發送中（等待回應）
FULFILLED 成功獲取數據（走 .then）
REJECTED 出現錯誤 / 斷網（走 .catch）

🔗 2. Fetch 的基本語法與鏈結 (40 min)

fetch() 執行後會回傳一個 Promise。最關鍵的概念是：它需要兩次等待。

📡 第一次等待：Response Header

當伺服器回傳狀態（如 200 OK）時，Promise 就會結束進入 Fulfilled 狀態。這時我們拿到的是 Response 物件，包含標頭資訊。

fetch(url).then(response => { ... })

📦 第二次等待：Data Body

數據流（Body）需要時間下載與解析（如轉成 JSON）。這也是一個非同步過程，所以 res.json() 會回傳第二個 Promise。

.then(res => res.json()).then(data => { ... })

🚀 線上任務：基礎鏈結練習

打開 DevTools Console，練習撰寫以下代碼，向一個公開 API 獲取資料：

fetch('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => response.json())
  .then(json => console.log(json));

✨ 3. Async / Await 的優雅化 (30 min)

雖然 .then() 已經很好，但當邏輯複雜時，async/await 能讓非同步代碼看起來像同步一樣易讀。

語法轉換：從 .then 到 Await

async function getData() {
  try {
    const res = await fetch(url);
    if (!res.ok) throw new Error('伺服器錯誤');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error('發生錯誤：', err);
  }
}

💡 關鍵提醒：檢查 response.ok

fetch 很頑強！即使伺服器傳回 404 或 500，它不會進入 catch。只有在斷網或 DNS 解析失敗時才會進入 catch。因此，務必檢查 response.ok。

作業指引 (Assignment Guide)

【作業 2-1】JSON 數據獲取與動態渲染

  • 角色扮演： 前端開發實習生 (Frontend Intern)。
  • 執行方式： 使用 fetch() 從公共 API (如 JSONPlaceholder) 獲取使用者清單。透過 forEach 遍歷數據，並使用 innerHTML 或 createElement 將姓名與 Email 動態顯示在網頁列表中。
  • 評分重點 : 成功解析 JSON 數組，且頁面能正確渲染出所有項目，並包含基礎的 CSS 樣式。

【作業 2-2】POST 請求：提交自定義數據

  • 角色扮演： 網頁整合工程師 (Web Integration Engineer)。
  • 執行方式： 實作一個「留言板」功能。當使用者按下提交按鈕時，將輸入框內容封裝成 JSON 物件，並透過 fetch(url, { method: 'POST', body: ... }) 傳送至模擬端點。
  • 評分重點 : 正確設定 POST 方法與 Content-Type 標頭，且能從伺服器回應中取出成功狀態。

【作業 2-3】網路異常處理：斷網與逾時保護

  • 角色扮演： 使用者體驗工程師 (UX Engineer)。
  • 執行方式： 為 Fetch 請求加入「強韌性」設計。實作 try...catch 結構處理斷網異常，並使用 AbortController 設定 5 秒超時，若請求逾時則顯示友好的錯誤提示訊息。
  • 評分重點 : 邏輯完整，能針對不同異常場景（如 404 vs 斷網）提供差異化的 UI 回饋。

B. 參考範例代碼 (Reference Example Code)

main.js (Async Fetch)

async function loadUsers() {
  const container = document.getElementById('list-box');
  try {
    const res = await fetch('https://api.example.com/users');
    if (!res.ok) throw new Error('API 無法連線');
    const data = await res.json();
    
    data.forEach(user => {
      const item = document.createElement('div');
      item.textContent = user.name;
      container.appendChild(item);
    });
  } catch (err) {
    container.innerHTML = `${err.message}`;
  }
}

C. 自動評分 (Autograding) 的設計

  • 關鍵字檢測：： 使用正規表示法 (Regex) 檢查程式碼中是否正確包含 async, await 以及 fetch。
  • 狀態捕捉驗證：： 透過測試代碼確認是否包含 try-catch 容錯區塊，避免網頁在異常時無反應。
