🎨 1. Canvas 畫布：數位繪圖的物理基礎 (20 min)

座標系統：理解 Canvas 的原點 (0, 0) 位在左上角。$x$ 軸向右增加，$y$ 軸向下增加（這與傳統笛卡兒座標系不同）。

🖌️ Context (2D)

它是你的「畫筆」，所有的繪圖指令（如 fillRect, drawImage）都必須透過它發送。

📷 影像來源

學習如何將 <img> 標籤或 <video> 標籤的內容「捕捉」到 Canvas 中實施即時處理。

座標對比：

  • Web Canvas: y 軸向下為正
  • 常見數學: y 軸向上為正

🔍 學習任務

在紙上畫出一個 $320 \times 240$ 的網格，標註出 $(160, 120)$ 的位置，並思考為什麼高頻率重繪（如 30 FPS）需要 requestAnimationFrame。

🧬 2. ImageData：揭開像素的秘密 (25 min)

RGBA 陣列：當我們執行 getImageData() 時，會得到一個超長的一維陣列 Uint8ClampedArray。

儲存規律：每 4 個數字代表一個像素

R G B A ...
  • R, G, B: 紅綠藍 (0-255)
  • A: 透明度 (Alpha, 0-255)

📐 索引公式 (Index Formula)

學習如何在二維影像中定位一維陣列的索引：

$Index = (y \times Width + x) \times 4$

* 乘以 4 是因為每個像素包含 R, G, B, A 四個分量。

🧮 3. 濾波與變換的數學邏輯 (15 min)

影像處理核心概念

  • 灰階化：將 R、G、B 取平均值。
  • 水平翻轉 (Mirroring)：座標變換的應用。
  • 二進位影像 (Blob)：學習如何將 Blob 繪製到畫布。

效能與優化

<img> 標籤與 <canvas> 繪圖的效能差異。

線上任務：

撰寫 JS，實作在影像上方疊加「十字準星」或「網格線」。

🚀 延伸思考

為什麼我們不直接使用 CSS 濾鏡處理，而是要手動操作 ImageData？

提示：當我們需要計算影像中的「紅點座標」或「追蹤物體」時，CSS 濾鏡能提供這些數據嗎？

進階課程 2 - 單元 2 | Canvas 影像處理

作業指引 (Assignment Guide)

作業 2-1：即時濾鏡開發師：灰階與反相

  • 角色扮演： 前端影像工程師。
  • 執行方式： 實作一個循環，每秒 30 次從 <video> 擷取畫面至 Canvas，並對 ImageData.data 進行線性變換。
  • 評分重點： 能否正確計算索引 Index = (y * Width + x) * 4 並避免索引越界。

作業 2-2：視覺準星：動態標註系統

  • 角色扮演： UI/UX 互動設計師。
  • 執行方式： 在影像 Canvas 上疊加第二層透明 Canvas，繪製會隨位置移動的十字準星，並顯示即時 RGBA。
  • 評分重點： 代碼結構是否將影像處理與標註繪製功能解耦。

作業 2-3：影像數據鏡像與 ROI 提取

  • 角色扮演： 視覺導航工程師。
  • 執行方式： 實作影像水平反轉，並從畫面中提取 100x100 的局部區域 (ROI) 放大顯示。
  • 評分重點： 座標空間變換公式的正確應用。

灰階濾鏡邏輯 (C-style optimized)

for (let i = 0; i < data.length; i += 4) {
    const avg = (data[i] + data[i+1] + data[i+2]) / 3;
    data[i] = data[i+1] = data[i+2] = avg; 
}

C. 自動評估 (Autograding) 的設計

  • 像素對比： 將處理過的 ImageData 與標準結果進行逐點位元對比。
  • 效能基準： 紀錄渲染一幀的時間，檢查是否維持在 33ms 以內。
