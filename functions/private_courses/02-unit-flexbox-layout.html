核心概念：容器與軸線

只要在父容器設定 display: flex;，它就變成了 Flex Container，裡面的子元素立刻變成乖巧的 Flex Items。

🔑 核心語法

↔️

主軸 (Main Axis)

決定排隊的方向。預設是水平向右 (`row`)。

↕️

交錯軸 (Cross Axis)

與主軸垂直的方向。決定元素如何"換行"或"堆疊"。

Module 2: Core Principles

對齊三劍客

1. justify-content

主軸上的分配。決定按鈕也是靠左、靠右，還是分散對齊。

space-between

2. align-items

交錯軸上的對齊。讓不同大小的按鈕在同一水平線居中。

center

3. flex-wrap

空間不夠時的策略。要硬擠在同一排，還是換行？

作業指引 (Assignment Guide)

作業 2-1：十字方向鍵 (幾何對稱挑戰)

  • 技術點： 巢狀容器（Nested Containers）的運用與軸線向性。
  • GitHub 優勢： 這是練習 Code Review 的好機會。你可以檢查學生是否為了達成對齊而使用了不必要的 margin，「抓出」那些沒用到 Flexbox 優勢的寫法（如手動推位置）。

作業 2-2：響應式佈局 (RWD 切換)

  • 技術點： flex-direction 隨螢幕寬度改變的邏輯判斷。
  • GitHub 優勢： 學生提交 PR 時，助教可以在 Files Changed 中直接對比不同中斷點（Breakpoint）下的 CSS 數值變動，確保其 flex-grow 或 flex-shrink 設定符合邏輯。

作業 2-3：間距與觸控優化 (精確控制)

  • 技術點： gap 屬性取代傳統 margin 的效率。
  • GitHub 優勢： 透過 **Commit History**，你可以看到學生如何微調 gap 數值，這反映了他們在不同模擬設備上測試佈局的過程。

在 GitHub Classroom 的實戰部署建議

建議範本倉庫設置如下：

A. 範本倉庫 (Template Repo) 結構

.
├── index.html       # 遙控器主頁面（HTML 框架已定）
├── style.css        # (重點) 學生需在此實作 Flexbox 佈局屬性
├── assets/          # 遙控器需要的圖示資產
└── README.md        # 教導如何使用 Chrome DevTools 模擬不同手機

B. 自動評分 (Autograding) 的設計

  • 屬性檢查： 自動掃描 CSS 文件，確認是否使用了 display: flex; 及對應的主軸對齊語句。
  • 佈局驗證： 利用簡單的 JS 腳本檢測關鍵元素（如 .d-pad）的子元素是否具備正確的 `offsetWidth`，以判定是否真正達成平分或對齊。

GitHub Classroom 詳細操作指南

第一步：準備範本倉庫 (Template Repo)

  1. 在 GitHub 建立新 Repo（名為 flexbox-remote-template）。
  2. 上傳基礎 HTML/CSS 框架。
  3. 在 Repo 設定中勾選 "Template repository"。

第二步：建立 Classroom 作業

  1. 進入 GitHub Classroom，建立新作業。
  2. 選擇「Individual Assignment」，並連結上述的範本倉庫。
  3. 重點設定：勾選 "Enable feedback pull requests"。這會自動建立一個 PR 供你留言與 Code Review。

第三步：學生端操作 (Flipped Workflow)

  1. 學生點擊作業連結，獲得個人專屬 Repo。
  2. 使用 VS Code Clone 到本地或直接在 GitHub Codespaces 開發。
  3. Push 代碼：只要學生 Push，GitHub 就會自動更新 PR 中的「檔案比對」。

第四步：教師回饋 (Review)

  1. 進入 GitHub Classroom 儀表板，點擊學生的 Repo 連結。
  2. 進入 Pull Requests 分頁，點擊 "Feedback" PR。
  3. 在 Files Changed 中，直接在代碼行旁點擊「+」號留言回饋（例如：「這裡的 justify-content 建議改用 space-around 會更平衡」）。
