基礎 10 - 單元 1

有限狀態機 (FSM)
日常裝置中的秘密邏輯

擺脫混亂的 if-else 嵌套，用「狀態」優化你的無人車控制架構。

🤖 1. 什麼是 FSM？ (20 min)

核心定義：一個系統在任何時間點，都處於「有限個狀態」中的其中一個。這能讓系統行為變得可預測且穩定。

狀態 (State)

例如「前進」、「停止」、「倒車」。系統目前「正在做什麼」。

事件 (Event)

觸發改變的條件，如「按下按鈕」、「感測器偵測到障礙物」。

轉移 (Transition)

從一個狀態變成另一個狀態的「動作」。

動作 (Action)

進入該狀態後要做的事（如：馬達轉動、變換 LED 顏色）。

🎨 2. 畫出狀態轉移圖 (30 min)

關鍵符號：圓圈代表狀態，箭頭代表轉移。這張圖將成為你編寫程式的「設計說明書」。

🎯 線上任務：繪製狀態圖

繪製一張狀態轉換圖（State Diagram），標出在什麼情況下（例如：按下按鈕、偵測到障礙物）系統會切換模式。

💻 3. 程式語法：switch...case 與 enum (20 min)

優點：FSM 在程式碼中通常不是用一大堆 if...else，而是用 switch 語法，這能讓邏輯更清晰、易於擴充。

💎 enum: 定義狀態名稱

使用 enum 來給狀態一個可讀的名字，避免在程式碼中使用 0, 1, 2 這種難以理解的「魔術數字」。

💎 switch: 事件處理中心

根據目前的狀態變數進行對應的跳轉。

// Arduino / C++ Version
enum State {
  IDLE, DRIVING, STOPPED
};

State currentState = IDLE;

void loop() {
  switch(currentState) {
    case IDLE:
      if (buttonPressed) currentState = DRIVING;
      break;
    case DRIVING:
      if (obstacle) currentState = STOPPED;
      break;
  }
}

核心任務：設計你的無人車邏輯

透過 FSM 結構，讓車子在複雜任務間切換得游刃有餘。

🕹️ 60 min

【作業 1-1】

「基礎控制狀態機」

  • 設計三個狀態：IDLE (待機)、REMOTE (手控)、AUTO (自律避障)。
  • 循環切換：按下網頁按鈕後循環切換。
  • 視覺反饋：不同狀態對應不同 LED 顏色（如 待機綠、手控藍、自律紅）。
🛡️ 60 min

【作業 1-2】

「避障邏輯狀態機」

  • 細分避障行為：FOR_SEARCH -> OBSTACLE_DETECTED -> BACKING -> TURNING。
  • 計時器結合：利用 millis() 設定轉向 500ms 後自動切回前進。
  • 解決混亂：用 FSM 取代傳統深層嵌套的 if-else 邏輯。
🆘 45 min

【作業 1-3】

「異常救援模式」

  • 新增 EMERGENCY_STOP 狀態。
  • 最高優先權：斷開藍牙或按下緊急鈕時，「強行轉移」到此狀態並鎖死馬達。
  • 重置機制：設計長按 3 秒重置回 IDLE 的邏輯。

作業指引 (Assignment Guide)

【作業 1-1】實作「基礎控制狀態機」 (三模切換)

  • 角色扮演： 韌體架構工程師 (Firmware Architect)。
  • 執行方式： 建立三個基礎狀態：IDLE (待機)、REMOTE (手控)、AUTO (自律)。透過網頁按鈕切換狀態，並結合板載 LED 顯示對應顏色（如綠、藍、紅）。
  • 評分重點 : 成功實作 enum 與 switch...case 結構，狀態切換流暢，且 LED 反饋即時。

【作業 1-2】實作「避障邏輯狀態機」 (含計時轉向)

  • 角色扮演： 自動駕駛演算法工程師 (Autonomous Driving Engineer)。
  • 執行方式： 細分避障行為為 SEARCH (前行搜尋) -> DETECTED (偵測障礙) -> TURN (轉向) 三個子狀態。利用 millis() 設定轉向 500ms 後自動切回搜尋狀態。
  • 評分重點 : 成功利用狀態機取代傳統 if-else 嵌套，邏輯清晰且能準確處理計時觸發的轉移。

【作業 1-3】異常處理：最高優先權「救援模式」

  • 角色扮演： 安全系統工程師 (Safety Systems Engineer)。
  • 執行方式： 新增 EMERGENCY 狀態。實作最高優先權切換機制：當按下物理緊急鈕或藍牙斷線時，「強行轉移」到此狀態鎖死馬達。需設計長按重置回 IDLE 的解鎖邏輯。
  • 評分重點 : 安全保護機制具備最高優先權，且重置邏輯嚴謹，無誤觸風險。

GitHub Classroom 詳細操作指南

第一步：準備範本倉庫 (Template Repo)

  1. 在 GitHub 建立新 Repo（名為 basic-10-fsm-template）。

  3. 在 Repo 設定中勾選 "Template repository"。

第二步：建立 Classroom 作業

  1. 進入 GitHub Classroom 建立「Individual Assignment」。
  2. 連結上述的範本倉庫，並勾選 "Enable feedback pull requests"。

第三步：學生端操作 (Flipped Workflow)

  1. 學生領取作業，獲得個人專屬 Repo。
  2. 繪製狀態轉移圖 (State Diagram)，將紙本邏輯實作為 switch...case 語句。
  3. 完成三模切換與避障邏輯後 Push 代碼 提交 PR。

第四步：教師回饋 (Review)

  1. 進入 Feedback PR 的 Files Changed。
  2. 重點審查學生是否漏掉 break 語句 (Fall-through bug) 以及轉移條件的嚴謹度。

main.cpp (FSM Skeleton)

enum State { IDLE, AUTO };
State mode = IDLE;
void loop() {
  switch(mode) {
    case IDLE: if(btn) mode = AUTO; break;
    case AUTO: avoidObstacles(); break;
  }
}
