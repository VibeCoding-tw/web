基礎 10 - 單元 1

有限狀態機 (FSM)
日常裝置中的秘密邏輯

擺脫混亂的 if-else 嵌套，用「狀態」優化你的無人車控制架構。

🤖 1. 什麼是 FSM？ (20 min)

核心定義：一個系統在任何時間點，都處於「有限個狀態」中的其中一個。這能讓系統行為變得可預測且穩定。

狀態 (State)

例如「前進」、「停止」、「倒車」。系統目前「正在做什麼」。

事件 (Event)

觸發改變的條件，如「按下按鈕」、「感測器偵測到障礙物」。

轉移 (Transition)

從一個狀態變成另一個狀態的「動作」。

動作 (Action)

進入該狀態後要做的事（如：馬達轉動、變換 LED 顏色）。

🎨 2. 畫出狀態轉移圖 (30 min)

關鍵符號：圓圈代表狀態，箭頭代表轉移。這張圖將成為你編寫程式的「設計說明書」。

🎯 線上任務：繪製狀態圖

繪製一張狀態轉換圖（State Diagram），標出在什麼情況下（例如：按下按鈕、偵測到障礙物）系統會切換模式。

💻 3. 程式語法：switch...case 與 enum (20 min)

優點：FSM 在程式碼中通常不是用一大堆 if...else，而是用 switch 語法，這能讓邏輯更清晰、易於擴充。

💎 enum: 定義狀態名稱

使用 enum 來給狀態一個可讀的名字，避免在程式碼中使用 0, 1, 2 這種難以理解的「魔術數字」。

💎 switch: 事件處理中心

根據目前的狀態變數進行對應的跳轉。

// Arduino / C++ Version
enum State {
  IDLE, DRIVING, STOPPED
};

State currentState = IDLE;

void loop() {
  switch(currentState) {
    case IDLE:
      if (buttonPressed) currentState = DRIVING;
      break;
    case DRIVING:
      if (obstacle) currentState = STOPPED;
      break;
  }
}

核心任務：設計你的無人車邏輯

透過 FSM 結構，讓車子在複雜任務間切換得游刃有餘。

🕹️ 60 min

【作業 1-1】

「基礎控制狀態機」

  • 設計三個狀態：IDLE (待機)、REMOTE (手控)、AUTO (自律避障)。
  • 循環切換：按下網頁按鈕後循環切換。
  • 視覺反饋：不同狀態對應不同 LED 顏色（如 待機綠、手控藍、自律紅）。
🛡️ 60 min

【作業 1-2】

「避障邏輯狀態機」

  • 細分避障行為：FOR_SEARCH -> OBSTACLE_DETECTED -> BACKING -> TURNING。
  • 計時器結合：利用 millis() 設定轉向 500ms 後自動切回前進。
  • 解決混亂：用 FSM 取代傳統深層嵌套的 if-else 邏輯。
🆘 45 min

【作業 1-3】

「異常救援模式」

  • 新增 EMERGENCY_STOP 狀態。
  • 最高優先權：斷開藍牙或按下緊急鈕時，「強行轉移」到此狀態並鎖死馬達。
  • 重置機制：設計長按 3 秒重置回 IDLE 的邏輯。
