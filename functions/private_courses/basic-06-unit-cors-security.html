CORS 與 ESP32 安全基礎

🛑 1. 什麼是 CORS？瀏覽器的邊界感 (30 min)

CORS (Cross-Origin Resource Sharing) 跨來源資源共用，是一套讓瀏覽器決定是否允許網頁讀取另一個網域資料的機制。

同源政策 (Same-Origin Policy)

為了防止惡意網站偷取你的私人資料（如銀行登入資訊），瀏覽器預設禁止「網域 A」的腳本去讀取「網域 B」的回傳內容。

判定標準：協議 (http/https)、網域 (domain)、連接埠 (port) 只要有一個不同，就是「跨來源」。

同源判定練習

http://vibe.com:80/a.js
http://vibe.com:80/b.js SAME
https://vibe.com
http://vibe.com CROSS
http://vibe.com:3000
http://vibe.com:8080 CROSS

🙋‍♂️ 2. 預檢請求 (Preflight Request) (20 min)

當你發送「複雜」請求（如 POST 或自定義 Header）時，瀏覽器會像個禮貌的小朋友，先去敲門詢問。

📩 OPTIONS 請求

瀏覽器會自動發送一個 OPTIONS 請求，詢問伺服器：「我可以進來嗎？我打算用這幾種方法和標頭」。

🔑 伺服器的許可證

伺服器必須回傳 Access-Control-Allow-Origin。如果伺服器說「不行」或是沒回，瀏覽器就會攔截請求。

🚀 關鍵代碼：ESP32 配置回應標頭

在 ESP32 的 Web Server 中，你需要手動加入許可標頭：

server.on("/api/control", HTTP_POST, [](AsyncWebServerRequest *request){
  request->send(200, "text/plain", "OK");
});

// 設定全域 CORS 許可 (這僅限開發測試)
DefaultHeaders::Instance().addHeader("Access-Control-Allow-Origin", "*");
DefaultHeaders::Instance().addHeader("Access-Control-Allow-Methods", "GET, POST, OPTIONS");
DefaultHeaders::Instance().addHeader("Access-Control-Allow-Headers", "Content-Type");

🔐 3. 基礎安全性防禦 (30 min)

理解為什麼「能動」並不代表「安全」。

HTTPS 的必要性

在 BLE 控制或傳輸敏感資料時，加密傳輸是唯一的選擇。沒有 HTTPS，所有的指令都是「裸奔」，中間人可以輕易攔截。

API Key 的保護

為什麼不能直接將 API 密鑰寫在前端 JS 代碼中？因為所有人按 F12 都能看到。前端是「不可信」的區域。

防禦性編程思維

  • 01 權限最小化：只開放必要的 Origins 給你的 API。
  • 02 資料遮蔽：在前端處理顯示時，隱藏部分 Token 或密碼。
  • 03 指令簽章：利用雜湊 (Hash) 確保指令在傳輸過程中未被竄改。

講師手冊 (Instructor Guide)

教學重點 (Teaching Points)

  • 解釋 CORS 實際上是瀏覽器的安全保護機制，而非伺服器的錯誤。
  • 討論 Access-Control-Allow-Origin: * 在開發階段與生產階段的利弊。
  • 介紹為什麼 IoT 裝置即使在區網內也需要基礎的身份驗證。

學生常見問題 (Common Issues)

  • 在瀏覽器報 CORS 錯誤時，死命修改 ESP32 的 URL 而非修改 Response Header。
  • 不理解 OPTIONS 請求（Preflight）的意義，導致複雜請求（如自定義 Header）一直失敗。
  • 在不支援 HTTPS 的 ESP32 上嘗試發送機密數據。

評分標準 (Grading)

  • 優 (90-100): 能正確設定 CORS Allow-Headers 並通過 Preflight 驗證，且對安全性有深刻見解。
  • 可 (70-89): 基礎功能正常但設定過於寬鬆（如全部使用 *），或對 Preflight 原理理解不足。
  • 待加強 (Below 70): 無法解決 CORS 攔截問題，導致網頁與硬體通訊中斷。

在 GitHub Classroom 的實戰部署建議

A. 範本倉庫 (Template Repo) 結構

.
├── src/             # 包含 ESPAsyncWebServer CORS 設定代碼的 main.cpp
├── README.md        # 實驗報告：跨網域連線錯誤截圖與修復後的標頭日誌
└── .github/         # 自動評核配置

B. 自動評估 (Autograding) 的設計

  • 標頭屬性檢測： 模擬瀏覽器發送 OPTIONS 請求，驗證回傳的 Header 是否包含 Access-Control-Allow-Origin。
  • 安全性檢核： 掃描程式中對於來源 Origin 的處理邏輯，獎勵具備動態過濾白名單（而非僅使用 *）的實作。
