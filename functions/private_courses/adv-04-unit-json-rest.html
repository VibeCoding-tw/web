作業指引 (Assignment Guide)

作業 3-1：標準化封包產生器

  • 角色扮演： 後端 API 開發者。
  • 執行方式： 整合多種感測器數據（如 $X, Y$ 質心座標、電量、運行狀態），設計一個嚴謹的巢狀 JSON 結構。實作 `/api/telemetry` GET 接口，模擬網頁端請求時回傳當前所有狀態的 JSON 封包。
  • 評分重點： JSON 結構是否符合邏輯且具備良好的縮排與欄位命名。

作業 3-2：指令接收器 (POST)

  • 角色扮演： 系統整合工程師。
  • 執行方式： 在 ESP32 上實作 `/api/move` 的 POST 接口，接收來自網頁端的 JSON 指令（如 `{"action": "turn", "angle": 45}`）。需解析封包並根據指令內容控制硬體動作。
  • 評分重點： 代碼中是否包含對 Payload 的合法性檢查（如缺少 Key 時的回報機制）。

作業 3-3：API 效能與壓力測試

  • 角色扮演： 雲端效能優化工程師。
  • 執行方式： 對比「冗餘版」與「精簡版」JSON 封包的大小，並使用迴圈進行連續高頻 (10Hz) 請求，記錄 ESP32 的回應延遲 (RTT) 與記憶體溢位風險。
  • 評分重點： 實驗報告是否能指出 JSON 序列化過程中 CPU 與 Bandwidth 的瓶頸。
