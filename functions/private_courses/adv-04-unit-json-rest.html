作業指引 (Assignment Guide)

作業 3-1：標準化封包產生器

  • 角色扮演： 後端 API 開發者。
  • 執行方式： 整合感測器數據設計巢狀 JSON 結構，實作 /api/telemetry GET 接口。
  • 評分重點： JSON 結構邏輯性與欄位命名的語義化。

作業 3-2：指令接收器 (POST)

  • 角色扮演： 系統整合工程師。
  • 執行方式： 實作 /api/move POST 接口，接收並解析 JSON 指令以控制硬體。
  • 評分重點： 代碼中是否包含 Payload 合法性檢查機制。

作業 3-3：API 效能與壓力測試

  • 角色扮演： 雲端效能優化工程師。
  • 執行方式： 對比冗餘與精簡封包大小，進行高頻請求測試並記錄延遲。
  • 評分重點： 實驗報告是否能指出序列化過程中的瓶頸。
