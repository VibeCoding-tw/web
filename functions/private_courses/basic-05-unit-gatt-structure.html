🏛️ 1. 藍牙的階層美學：GATT 結構 (20 min)

GATT (Generic Attribute Profile) 定義了兩台藍牙設備如何交換數據。你可以把它想像成一個組織嚴密的圖書館：

📦 Profile (規範)

設備的角色定義。例如：「一台無人車」。

📂 Service (服務)

功能分類的資料夾。例如：「動力系統」、「遙測數據」。

📄 Characteristic (特徵)

最基本的數據檔案。例如：「馬達速度值」、「電池電量」。

💡 學習任務

想像你要設計一台「智慧燈泡」，請嘗試列出它可能包含哪些 Service 與 Characteristic？

🆔 2. 身份證字號：UUID (15 min)

每一個 Service 與 Characteristic 都需要一個唯一的身份證，這就是 UUID。

  • 唯一識別碼：128-bit 的結構確保全世界的藍牙設備功能不會互相衝突。
  • 標準 UUID：藍牙聯盟定義好的功能（如：心率計：0x180D）。
  • 自定義 UUID：我們開發無人車時自行生成的 UUID，用於特殊功能。
// 範例自定義 UUID
#define SERVICE_UUID "4fafc201-1fb5-459e-8fcc-c5c9c331914b"
#define CHARACTERISTIC_UUID "beb5483e-36e1-4688-b7f5-ea07361b26a8"

🔄 3. 數據交換的四種方式 (25 min)

📥 Read (讀取)

手機主動向車子索取數據。適用於不常變動的狀態（如：韌體版本）。

📤 Write (寫入)

手機發送指令控制車子。例如：發送「前進」指令給馬達特徵值。

🔔 Notify (通知)

車子主動推播。例如：電量低於 10% 時自動告知手機。最高效的方式。

✅ Indicate (指示)

類似 Notify，但需要手機回傳確認收到。更安全但較慢。

作業指引 (Assignment Guide)

【作業 1-1】「定義你的無人車服務」

  • 角色扮演： 系統架構師 (System Architect)。
  • 執行方式： 在 ESP32 上規劃並實作一個自定義的 GATT Service（如 Motor_Control）。生成 128-bit UUID，並成功透過手機端 App 掃描出該服務。
  • 評分重點 : UUID 符合規範且不與範例衝突，服務在移動端能被正確列出。

【作業 1-2】「雙向對話測試」：Read & Write

  • 角色扮演： 韌體整合工程師 (Firmware Integration Engineer)。
  • 執行方式： 在同一個 Service 下管理讀取與寫入屬性。實作一個可讀取的「設備運作時間」特徵值，以及一個可寫入控制板載 LED 燈效的特徵值。
  • 評分重點 : 數據格式轉換正確，能在手機端即時變更 ESP32 硬體狀態。

【作業 1-3】「即時遙測：實作 Notify」

  • 角色扮演： 資料遙測工程師 (Data Telemetry Engineer)。
  • 執行方式： 建立一個模擬「轉速感測器」的特徵值，開啟 NOTIFY 屬性。當感測值更新時，ESP32 必須主動推播數據至手機，不需手機主動發問。
  • 執行方式：： 建立一個模擬「轉速感測器」的特徵值，開啟 NOTIFY 屬性。當感測值更新時，ESP32 必須主動推播數據至手機，不需手機主動發問。
  • 評分重點 : 成功配置通知描述符 (Descriptor)，且推播數據流連貫穩定。

B. 參考範例代碼 (Reference Example Code)

main.cpp (GATT 多層級架構)

#include <BLEDevice.h>

void setup() {
  BLEDevice::init("GATT_SERVER");
  BLEServer *pServer = BLEDevice::createServer();
  
  // Service 1: 遙控系統
  BLEService *pControl = pServer->createService(SERVICE_CONTROL_UUID);
  pControl->createCharacteristic(CH_SPEED_UUID, BLECharacteristic::PROPERTY_WRITE);
  pControl->start();
  
  // Service 2: 電池系統
  BLEService *pBattery = pServer->createService(SERVICE_BATTERY_UUID);
  pBattery->createCharacteristic(CH_LEVEL_UUID, BLECharacteristic::PROPERTY_NOTIFY);
  pBattery->start();
}

C. 自動評分 (Autograding) 的設計

  • 層級結構檢測：： 透過測試代碼自動掃描 GATT 樹狀圖，驗證 Service UUID 的數量與特徵值的歸屬關係是否正確。
  • UUID 規範檢核：： 確保學生使用的是隨機生成的 128-bit UUID，而非標準保留 ID。
