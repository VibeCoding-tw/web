Web Speech API：
開啟網頁的「聽覺與讀稿」能力

利用 JavaScript 原生 API 實現語音辨識與合成，探索雲端分析與本地引擎的權衡。

🏛️ 1. Web Speech API 的兩大支柱 (30 min)

SpeechRecognition (語音辨識)

將聲音轉化為文字（Speech-to-Text）。

  • 原理：擷取音訊後傳送到伺服器（如 Google 雲端）分析。
  • 限制：高度依賴網路，且不同瀏覽器支援度差異大。

SpeechSynthesis (語音合成)

將文字轉化為聲音（Text-to-Speech）。

  • 原理：利用作業系統內建語音引擎，不一定需要網路。

🧩 學習任務

在瀏覽器控制台 (Console) 嘗試簡單的 window.speechSynthesis.speak() 指令，讓電腦開口說話。

⚡ 2. 事件驅動與非同步生命週期 (30 min) 

recognition.onstart = () => {
  console.log('系統開始監聽...');
};

recognition.onresult = (event) => {
  const result = event.results[0][0].transcript;
  console.log('辨識結果:', result);
};

recognition.onerror = (e) => {
  console.error('發生錯誤:', e.error);
};
  • ● 識別事件：理解 onstart、onresult、onerror 與 onend 的順序。
  • ● 連續識別 (Continuous)：為什麼遙控車需要開啟 interimResults（中間結果），以達到即時反應？

📖 學習任務

閱讀官方文件，理解為什麼辨識過程會因為短暫的沉默而自動關閉，以及如何透過 continuous = true 設定解決。

🔒 3. 安全與隱私限制 (20 min)

  • ● HTTPS 要求：Web Speech API 必須在安全連線下運作，否則瀏覽器會拒絕權限。
  • ● 使用者授權：瀏覽器如何保護麥克風權限？這對「自動化」監聽系統有什麼影響？

⚠️ 重點提醒

理解語音識別在「連續監聽」與「關鍵字觸發」間的權衡，以及如何處理語句權重 (Weighting)。

核心實驗室：語音控制實戰

將語音指令轉化為精確的硬體動作，打造智慧無人車。

⌨️ 60 min

【作業 3-1】聲控指令集

任務內容：在 BLE 控制介面中整合語音辨識功能。

  • 關鍵字過濾：當結果包含「前進」、「左轉」時觸發函式
  • 模糊匹配：使用者說「請停下來」時應識別出「停」
🔊 60 min

【作業 3-2】噪音對抗

任務內容：優化在多人吵雜環境下的辨識成功率。

  • 信心指標 (Confidence)：顯示分數，低於 0.8 則忽略
  • 語音回饋：透過 SpeechSynthesis 回應「收到，開始前進」
🔄 45 min

【作業 3-3】FSM 整合

任務內容：利用語音切換無人車的工作模式。

  • 模式切換：說出「進入自動模式」時 UI 同步切換
  • 異常重啟：當 onend 觸發時自動重啟辨識
