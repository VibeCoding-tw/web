🏛️ 1. Web Speech API 的兩大支柱 (30 min)

SpeechRecognition (語音辨識)

將聲音轉化為文字（Speech-to-Text）。

  • 原理：擷取音訊後傳送到伺服器（如 Google 雲端）分析。
  • 限制：高度依賴網路，且不同瀏覽器支援度差異大。

SpeechSynthesis (語音合成)

將文字轉化為聲音（Text-to-Speech）。

  • 原理：利用作業系統內建語音引擎，不一定需要網路。

  console.error('發生錯誤:', e.error);

作業指引 (Assignment Guide)

作業 3-1：「聲控指令集」開發與模糊匹配

  • 角色扮演： 介面開發工程師。
  • 執行方式： 實作 `SpeechRecognition` 功能，建立一個關鍵字字典（如："前進", "停止", "左轉"）。使用字串包含 (`includes`) 邏輯處理使用者的自然口語。
  • 評分重點： 能否正確處理模糊指令，例如當使用者說「請快點停下來」時，程式能否精準抓到「停」並觸發正確的馬達動作。

作業 3-2：環境噪音下的「信心分數」過濾器

  • 角色扮演： 語音演算法工程師。
  • 執行方式： 在網頁介面上即時顯示語音辨識的 `confidence` 分數。實作一個門檻 (Threshold) 判斷，低於 0.8 時程式會回應「聽不清楚，請再說一次」。
  • 評分重點： 代碼中對 `onresult` 事件的處理是否嚴謹，以及能否透過 `SpeechSynthesis` 給予流暢的使用者回饋。

作業 3-3：聲控狀態機 (Voice FSM) 整合

  • 角色扮演： 系統整合專家。
  • 執行方式： 將語音指令與無人車的 FSM 深度串接。例如：說出「進入自動模式」時，UI 與車體狀態同步切換至 `AUTO_NAV`。
  • 評分重點： 處理「連續辨識 (Continuous)」的生命週期，確保辨識器不會因為短暫沉默而中斷，並在 `onend` 時自動重啟辨識。
