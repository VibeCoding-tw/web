為何電腦看不懂紅色：
HSV 的故事

深入探討 HSV 相比於 RGB 在變動光影環境下的穩定性，學習如何定義精準的顏色區間。

💔 1. RGB 的致命傷：光影耦合 (20 min)

問題現象：在 RGB 空間中，如果你調暗燈光，$R, G, B$ 三個數值會同時下降。這意味著對於電腦來說，「深紅色」與「明亮的粉紅色」在數學特徵上極度混亂。

// RGB 耦合示例

亮光下: (255, 100, 100) -> 淺紅

陰影下: (80, 20, 20) -> 深紅

三個維度都在動，難以設定穩定的門檻值。

🔍 學習任務：光影觀察

觀察同一張物體在「陽光下」與「陰影下」的 RGB 數值變化。

陽光下 (R,G,B): 220, 45, 45

陰影下 (R,G,B): 110, 22, 22

理解為什麼簡單的 RGB 門檻值無法在戶外穩定追蹤目標。

📐 2. HSV：幾何上的「分而治之」 (30 min)

H

Hue (色相)

代表「是什麼顏色」。它是一個角度（$0^\circ \sim 360^\circ$），是視覺追蹤最重要的參數。

S

Saturation (飽和度)

顏色的純度。

V

Value (亮度)

光的強弱。

數學優勢

在 HSV 中，只要顏色不變，不論光照如何變化（V 變動）或顏色變淡（S 變動），H 值通常保持穩定。

⭕ 3. 色環的連續性與斷點 (10 min)

🧠 預習挑戰：紅色的斷點

紅色的 H 值大約在 $0^\circ$（或 $360^\circ$）。思考在寫程式時，如果目標是紅色，門檻值該如何跨越這個「斷點」？

0° / 360°

核心實驗室：HSV 色彩攻防戰

採用 Git Workflow：透過 GitHub Classroom 提交色彩追蹤報告與代碼 PR。

  • 數據紀錄：紀錄強光與暗處時的 RGB 與 HSV 均值。
  • 變異分析：計算波動程度，證明 H 值的穩定性。

作業指引 (Assignment Guide)

進階 6 - HSV 顏色空間的數學優勢

3-1 色彩壓力測試：RGB vs. HSV

角色扮演： 光學感測工程師。

執行步驟：

  1. 準備固定顏色的目標物（如小紅球）。
  2. 在強光與陰影下紀錄 RGB 及 HSV 分量。
  3. 計算環境變化下的數值變異係數，對比穩定性。

評分重點： 數據是否能證明 H 值在不同光影下的高一致性。

3-2 動態 HSV 遮罩 (Mask) 產生器

角色扮演： 視覺演算法開發者。

執行步驟：

  1. 撰寫 rgbToHsv() 轉換函式。
  2. 建立 HSV 區間滑桿，即時產生黑白遮罩以過濾特定色塊。
  3. 處理色相環「紅色斷點」（360/0 跨度）的過濾邏輯。

評分重點： 遮罩產出的精確度，以及對紅色邊界邏輯的處理。

3-3 多色追蹤與優先權判定

角色扮演： 智慧交通系統設計師。

執行步驟：

  1. 定義紅、綠兩組 HSV 門檻。
  2. 實作邏輯：當紅/綠同時出現時，依「優先權」決定發送給車輛的導航指令。

評分重點： 多目標偵測的穩定性及決策邏輯的清晰度。
