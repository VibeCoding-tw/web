基礎 8 - 單元 3

曲線優化：
線性 vs. 指數響應

掌握演算法如何彌補人類肌肉控制的不足，打造完美手感。

📏 1. 線性響應 (Linear Response) (20 min)

定義：輸出與輸入成正比（\(y = x\)）。

優點：直觀預測

搖桿推一半，馬達就出一半的力。反應速度最快，沒有任何延遲感。

缺點：中心太敏感

在中心點附近的微調非常困難，對於需要精細動作（如窄路穿梭）的場景，操作感會顯得太過「敏感」。

Linear: \(f(x) = x\)

📈 2. 指數響應 (Exponential Response) (40 min)

定義：輸出隨輸入的次方增加（通常使用三次方 \(y = x^3\) 以保持正負號）。

☁️ 軟中心 (Soft Center)

中心區域的斜率較平緩，容許大幅度的手感微調。適合精細的轉彎。

💥 暴力尾段 (Punchy End)

當搖桿推到底時，數值會快速攀升至最大值，保有爆發力。

\(f(x) = x^3\)

紫線：指數 / 虛線：線性

🎯 線上任務

在 Desmos 或 Excel 繪製「輸入對輸出」的曲線圖。觀察當輸入為 0.5 時，線性輸出是 0.5，而三次方輸出是多少？（答案：0.125）

🌑 3. 為什麼需要「死區 (Deadzone)」？ (20 min)

硬體限制：便宜的搖桿在放手時可能無法精確回到 0（可能停在 0.02），導致車子在沒動手時也會慢慢滑行。

解決方案：加入小平台

在函數中加入一個小判斷，讓低於某個數值（如 0.1）的輸入一律強制輸出為 0。

// Deadzone Logic
if (Math.abs(value) < 0.1) {
  return 0;
}

核心任務：打造職人級手感

透過混合曲線調整，找到專屬於你的「駕駛風格」。

🔢 60 min

【作業 3-1】

「混合映射函式」

實作公式：

Output = (1 - k)·x + k·x³

讓學生透過調整比例參數 k (0~1)，觀察曲線從線性過渡到指數的過程。

🏎️ 60 min

【作業 3-2】

「窄路繞錐」測試

設置極窄 S 彎道。記錄使用「線性」與「指數」模式下的碰撞次數。體驗指數響應如何協助你進行微幅的方向修正。

🔘 45 min

【作業 3-3】

「動態模式切換」

在 UI 上增加切換開關，實作「精準模式 (Expo)」與「競速模式 (Linear)」的即時切換邏輯。

講師手冊 (Instructor Guide)

教學重點 (Teaching Points)

  • 人體工程學與控制：解釋為什麼人類手指在微小控制時需要更平緩的曲線（而非線性）。
  • 冪函數 (Power Functions) 的應用：$y = x^n$ 在控制率中的實際效果。
  • 解析度與平滑度：討論曲線計算對馬達 PWM 輸出解析度的影響。

學生常見問題 (Common Issues)

  • 計算過程中遺失正負號：直接對輸入值取平方會導致負向控制變成正向。
  • 係數設定過大，導致車子在小範圍操作時反應過於遲鈍或突然暴衝。
  • 浮點數運算的效能疑慮（解釋 ESP32 FPU 的處理能力）。

評分標準 (Grading)

  • 優 (90-100): 指數曲線實作精確，能處理正負號，且切換靈敏度時過渡平滑。
  • 可 (70-89): 曲線功能正常，但代碼邏輯較為混亂，或未考慮極致的微操體驗。
  • 待加強 (Below 70): 無法正確計算非線性響應，或導致數值溢出。

B. 自動評估 (Autograding) 的設計

  • 非線性驗證： 輸入 0.5 (中位)，檢查輸出是否符合指數曲線特性（例如 $0.5^2 = 0.25$）。
  • 符號保存檢核： 驗證輸入為負值時，輸出是否正確保留負號而非變成正值。
