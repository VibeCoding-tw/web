基礎 8 - 單元 3

曲線優化：
線性 vs. 指數響應

掌握演算法如何彌補人類肌肉控制的不足，打造完美手感。

📏 1. 線性響應 (Linear Response) (20 min)

定義：輸出與輸入成正比（\(y = x\)）。

優點：直觀預測

搖桿推一半，馬達就出一半的力。反應速度最快，沒有任何延遲感。

缺點：中心太敏感

在中心點附近的微調非常困難，對於需要精細動作（如窄路穿梭）的場景，操作感會顯得太過「敏感」。

Linear: \(f(x) = x\)

📈 2. 指數響應 (Exponential Response) (40 min)

定義：輸出隨輸入的次方增加（通常使用三次方 \(y = x^3\) 以保持正負號）。

☁️ 軟中心 (Soft Center)

中心區域的斜率較平緩，容許大幅度的手感微調。適合精細的轉彎。

💥 暴力尾段 (Punchy End)

當搖桿推到底時，數值會快速攀升至最大值，保有爆發力。

\(f(x) = x^3\)

紫線：指數 / 虛線：線性

🎯 線上任務

在 Desmos 或 Excel 繪製「輸入對輸出」的曲線圖。觀察當輸入為 0.5 時，線性輸出是 0.5，而三次方輸出是多少？（答案：0.125）

🌑 3. 為什麼需要「死區 (Deadzone)」？ (20 min)

硬體限制：便宜的搖桿在放手時可能無法精確回到 0（可能停在 0.02），導致車子在沒動手時也會慢慢滑行。

解決方案：加入小平台

在函數中加入一個小判斷，讓低於某個數值（如 0.1）的輸入一律強制輸出為 0。

// Deadzone Logic
if (Math.abs(value) < 0.1) {
  return 0;
}

核心任務：打造職人級手感

透過混合曲線調整，找到專屬於你的「駕駛風格」。

🔢 60 min

【作業 3-1】

「混合映射函式」

實作公式：

Output = (1 - k)·x + k·x³

讓學生透過調整比例參數 k (0~1)，觀察曲線從線性過渡到指數的過程。

🏎️ 60 min

【作業 3-2】

「窄路繞錐」測試

設置極窄 S 彎道。記錄使用「線性」與「指數」模式下的碰撞次數。體驗指數響應如何協助你進行微幅的方向修正。

🔘 45 min

【作業 3-3】

「動態模式切換」

在 UI 上增加切換開關，實作「精準模式 (Expo)」與「競速模式 (Linear)」的即時切換邏輯。

作業指引 (Assignment Guide)

【作業 3-1】實作指數響應曲線 (Exponential Curve)

  • 角色扮演： 操縱工程師 (Control Engineer)。
  • 執行方式： 實作混合公式：Output = (1 - k)·x + k·x³。讓學生透過調整比例參數 k (0~1)，觀察曲線從線性過渡到三次方指數的過程。
  • 評分重點 : 成功實作非線性轉換，且能正確處理輸入的正負號（x³ 需保留輸入極性）。

【作業 3-2】「窄路繞錐」對比測試

  • 角色扮演： 專業試車員 (Test Driver)。
  • 執行方式： 設置極窄 S 彎道。分別使用「線性」與「指數」模式進行繞錐測試。記錄碰撞次數與完成時間。
  • 評分重點 : 能透過實測數據說明「軟中心 (Soft Center)」如何協助進行微幅方向修正。

【作業 3-3】動態模式切換 UI

  • 角色扮演： 車載 UI 設計師 (Automotive UI Designer)。
  • 執行方式： 在網頁控制介面上增加切換按鈕，實現「精準操縱 (Expo)」與「直覺反應 (Linear)」模式的即時熱切換。
  • 評分重點 : UI 互動流暢，模式切換無延遲，且具備清晰的目前狀態顯示。

GitHub Classroom 詳細操作指南

第一步：準備範本倉庫 (Template Repo)

  1. 在 GitHub 建立新 Repo（名為 basic-08-curves-template）。
  2. 上傳基礎通訊框架，但在控制邏輯中預留 applyCurve(float x) 虛擬函式。
  3. 在 Repo 設定中勾選 "Template repository"。

第二步：建立 Classroom 作業

  1. 進入 GitHub Classroom 建立「Individual Assignment」。
  2. 連結上述的範本倉庫，並勾選 "Enable feedback pull requests"。

第三步：學生端操作 (Flipped Workflow)

  1. 學生領取作業，獲得個人專屬 Repo。
  2. 實作線性、二次方、三次方等響應公式，並觀察數據分佈圖。
  3. 針對「低連線延遲」情境與「窄路穿梭」場景分別調校最佳係數後 Push 代碼。

第四步：教師回饋 (Review)

  1. 進入 Feedback PR 的 Files Changed。
  2. 針對學生的「係數動態調整感 (Responsiveness)」與「運算溢位處理」予以點評。

main.cpp (Mixed Response)

float input = getJoystickX();

C. 自動評分 (Autograding) 的設計

  • 數學模型檢核：： 使用特定輸入（如 0.2, 0.8），檢查輸出是否精確落在學生宣稱的曲線模型數值之上。
  • 極性一致性檢測：： 驗證學生在進行次方運算時，是否正確保留了負數輸入的回傳極性。
