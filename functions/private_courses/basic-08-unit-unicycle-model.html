基礎 8 - 運動學轉換

差速轉換數學：
Unicycle Model

將抽象的速度與轉向，解構成左右馬達的動力之源。

🚲 1. 什麼是 Unicycle Model？ (30 min)

核心概念：將車輛簡化為一個點，透過兩個參數來控制其運動模式。

v 線速度 (Linear Velocity)

代表車輛前進或後退的速度。在搖桿上，這通常對應到 Y 軸。

ω 角速度 (Angular Velocity)

代表車輛旋轉或轉向的速度。在搖桿上，這通常對應到 X 軸。

🎯 學習目標

理解為何搖桿的垂直位移控制速度，水平位移控制旋轉，這是建立直覺操控的基礎。

⚙️ 2. 差速驅動 (Differential Drive) 原理 (30 min)

物理結構：兩輪獨立驅動。要讓車轉向，左輪與右輪必須產生「速度差」。

基本混合公式

\[L = v - \omega\]
左輪轉速
\[R = v + \omega\]
右輪轉速

🧠 學習任務：腦力激盪

若 \(v=100\), \(\omega=50\)（代表前進中右轉），算出左、右輪各是多少？
（提示：左輪 50, 右輪 150）

🛑 3. 飽和度處理與比例縮放 (20 min)

⚠️ 問題：當數值溢出時？

若 \(v=255\) 且 \(\omega=100\)，算出的 \(R\) 會變成 \(355\)。但馬達最大 PWM 只有 \(255\)，直接裁切（Clipping）會導致轉向失靈。

預習概念：等比例縮放 (Normalization)

\[max\_val = \max(|L|, |R|, 255)\]
\[L_{final} = (L / max\_val) \times 255\]
\[R_{final} = (R / max\_val) \times 255\]

目標：即使在全速前進時，轉向指令依然能有效發揮作用，保持車輛的動態平衡。

作業指引 (Assignment Guide)

【作業 2-1】基礎差速混合器實作

  • 角色扮演： 軟體運動學工程師 (Software Kinematics Engineer)。
  • 執行方式： 編寫 mix(v, w) 函式。在教室進行原地旋轉測試與方向校正。觀察 \(v=0\) 時車子是否能精準原地打轉，並排除接線反向導致的誤型。
  • 評分重點 : 公式實作正確，左右馬達分量與 \(v, \omega\) 同步，操作感符合幾何直覺。

【作業 2-2】飽和度處理與比例縮放

  • 角色扮演： 控制系統優化專家 (Control System Optimization Expert)。
  • 執行方式： 實作等比例限幅演算法。當混控後的馬達數值超過 255 時，進行比例壓縮而非直接裁切。測試在全速前進時是否仍能保有良好的轉向能力。
  • 評分重點 : 演算法能有效防止全速前進時其中一輪飽和導致的控制失效，車體動態保持平衡。

【作業 2-3】轉向敏感度調教

  • 角色扮演： 賽道調校教練 (Track Tuning Coach)。
  • 執行方式： 加入 Sensitivity (K) 參數動態縮放 $\omega$。透過實驗找出最適合目前賽道寬度的 K 值，優化在 S 彎道中的循跡性能。
  • 評分重點 : 參數設計合理，能顯著改善在特定地形下的操控手感。

  2. 連結上述的範本倉庫，並勾選 "Enable feedback pull requests"。

  1. 學生領取作業，獲得個人專屬 Repo。

第四步：教師回饋 (Review)

  1. 進入 Feedback PR 的 Files Changed。
  2. 針對學生的「左右平衡係數」設計與「溢位壓縮演算法」合理性予以回饋。

B. 參考範例代碼 (Reference Example Code)

main.cpp (Kinematics Mixing)

float left = v - w;
float right = v + w;
float scale = max(1.0f, max(abs(left), abs(right)));
left /= scale; right /= scale; // Normalization

C. 自動評分 (Autograding) 的設計

  • 極限行為檢測：： 設定 \(v=1, \omega=1\)，驗證學生的演算法是否會導致其中一輪停擺或單純裁切 (Clipping)。
  • 原地旋轉驗證：： 給予 \(v=0, \omega \neq 0\)，檢查左右馬達 PWM 是否嚴格符合「一正一負且相等的絕對值」。
