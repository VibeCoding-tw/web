驅動程式與通訊埠原理

打通電腦與晶片之間的溝通橋樑。

🔌 1. 數位翻譯官：USB-to-UART 橋接晶片

通訊協定的衝突

電腦說的是 USB 語言（高速、複雜），就像現代人說得飛快的網路用語；而 ESP32 核心說的是 UART 語言（簡單、序列），就像傳統的電報。兩者直接對話是無法溝通的。

💻 USB (高速) ↔️ 橋接晶片 (翻譯) ↔️ 🤖 UART (序列)

橋接晶片的作用

開發板上一定有一顆不起眼的小晶片（常見型號如 CP2102, CH340, 或 FT232），它就是那個辛苦的口譯員，負責在兩者之間進行即時翻譯。

驅動程式的本質

驅動程式 (Driver) 其實就是給作業系統（Windows/Mac）看的「說明書」。如果沒安裝驅動，作業系統就不知道如何操作這顆橋接晶片，自然也無法和 ESP32 說話。

🚪 2. 什麼是通訊埠 (COM Port)？

邏輯抽象化

當驅動程式正確安裝後，作業系統會將偵測到的轉接晶片「虛擬化」為一個「COM 埠」（Windows 上是 COM3, COM4...；Mac/Linux 上則稱為 /dev/tty...）。這就像是為這個裝置開了一個專屬的「門牌號碼」。

唯一性與衝突

每個插入的設備都會獲得一個獨一無二的編號。如果編號衝突，或者驅動程式根本沒裝好（找不到門牌），電腦就無法將寫好的程式碼「倒」進晶片裡，這就是初學者最常遇到的上傳失敗原因。

> ls /dev/tty.*
/dev/tty.Bluetooth-Incoming-Port
/dev/tty.usbserial-0001
/dev/tty.wlan-debug

⏱️ 3. 序列通訊的核心參數：Baud Rate

波特率 (Baud Rate)

簡單來說，Baud Rate 定義了每秒傳輸多少個位元 (bits)。這是通訊雙方必須遵守的「語速」。

  • 9600：傳統、慢速，Arduino 預設。
  • 115200：ESP32 最常用的標準速率。
  • 921600：極速，上傳程式碼時常用。

同步的重要性

序列通訊是「非同步」的，沒有一條線專門傳送時鐘訊號。所以發送端（ESP32）與接收端（電腦）必須事先約定好相同的速率（如 115200）。

如果一方講太快（115200），另一方聽太慢（9600），你就會在螢幕上看到滿滿的亂碼 (Garbled Text)。

  • 作業 1-3 (30%)：撰寫實驗報告，分析 Baud Rate 設錯時出現亂碼的原因。

講師手冊 (Instructor Guide)

1. 教學重點 (Teaching Points)

  • 解釋 USB-to-UART 橋接晶片 (CP2102/CH340) 的物理角色。
  • 強調 Baud Rate 同步的重要性：發送端與接收端必須達成共識。
  • 引導學生理解串列通訊中的 TX/RX 對接概念。

2. 常見問題 (Common Issues)

  • 抓不到 COM 埠：通常是 USB 數據線品質不佳或驅動程式未安裝。
  • 序列監控視窗出現亂碼：99% 是代碼中的 `Serial.begin()` 與工具設定頻率不一致。
  • 裝置忙碌 (Busy)：檢查是否有其他軟體（如 Arduino IDE）佔用了同一個埠。

3. 評分標準 (Grading Criteria)

  • 基礎分：能正確開啟 Serial Monitor 並觀察到數據。
  • 進階分：能解釋 DTR/RTS 引腳對 ESP32 自動燒錄的影響。

B. 自動評估 (Autograding) 的設計

  • 核心配置檢核： 偵測程式中是否呼叫了 Serial.begin()，並驗證其參數是否為常用的 115200。
  • 資源相符性驗證： 透過編譯時間檢查 platformio.ini 中的 monitor_speed 設定是否與程式碼同步。

GitHub Classroom 詳細操作指南

第一步：準備範本倉庫 (Template Repo)

建立包含基礎 Serial 框架但「Baud Rate 設錯」的 Repo，於 GitHub 設定中勾選 "Template repository"。

第二步：建立 Classroom 作業

建立 Individual Assignment，連結範本並啟用 "Feedback pull requests" 以便對其硬體通訊協議的理解進行點評。

第三步：學生端操作 (Flipped Workflow)

學生領取作業，獲得個人專屬 Repo，排除驅動程式與硬體接口障礙，成功印出 Driver 狀態後 Push。

第四步：教師回饋 (Review)

進入 Feedback PR，針對學生的「通訊除錯思維 (Debugging Mindset)」與「基礎設施理論」設計予以回饋。
