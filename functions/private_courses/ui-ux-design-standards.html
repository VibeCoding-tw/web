UI/UX 設計規範：從視覺到體驗

不只是好看，更要好用。學習如何利用「翻轉教室」的概念，設計出直覺、高效且符合人體工學的 FPV 遙控介面。

1

拇指操作區與人體工學

在設計手機遙控介面時，**Thumb Zone（拇指熱區）** 是最關鍵的概念。了解雙手持握時，拇指能輕鬆觸及的範圍，決定了核心按鈕的佈局。

💡 關鍵思考

  • ➜ 為什麼「緊急停止」必須放在兩側下方最容易按到的地方？
  • ➜ 為什麼「設定選單」通常放在螢幕中央或上方較難觸及的區域？

2

視覺階層與認知負荷

⚠️

色彩編碼

顏色傳遞訊息，不需文字。紅色與黃色能瞬間抓住注意力，綠色則給人安全感。

📏

大小差異

最重要的功能應該最大。根據 Fitts's Law，目標越大、距離越近，操作所需時間越短。

Primary
🧠

資訊降噪

FPV 競速時，駕駛者沒有時間閱讀。只顯示當下必要的數據，隱藏次要資訊，降低認知負荷。

🎯 學習目標

預習如何減少操作者的認知負擔，讓使用者在高速飛行時不需要「思考」按鈕在哪裡，而是憑「直覺」操作。

按鈕狀態實驗室

Normal State

試著 Hover 與點擊左側按鈕，觀察顏色與陰影變化。

System Feedback
3

回饋設計 Loop

操作不只是單向的「輸入」。系統如何回應使用者的操作，決定了體驗的流暢度。

👀

視覺回饋

按下按鈕時的顏色變化、壓下效果，讓使用者確認「我有按到」。

📳

觸覺回饋 (Haptics)

手機震動是最直接的確認。在無法看螢幕的盲操情境下尤為重要。
