數據抖動：
馴服數位震顫

感測器數據為什麼會跳動？學習如何利用濾波演算法，從雜訊中提取真實訊號。

🤔 1. 為什麼數據會抖動？ (20 min)

💡 雜訊來源

環境光線波動、馬達電磁干擾 (EMI) 以及感測器內部的電子熱雜訊。這些外部與內部噪音會疊加在原始數據上。

📉 量化誤差

數位系統在將類比訊號轉換為數位值 (ADC) 時，由於解析度限制產生的微小階梯狀波動。

👀 學習任務

觀看一段「未經過濾的質心座標」影片，觀察數據點如何跳動，並思考這對馬達會造成什麼負擔（頻繁震動、過熱）。

🧼 2. 均值濾波器 (Moving Average) (25 min)

  • ● 核心概念：取最近 $N$ 筆數據的平均值，利用「時間換取穩定度」。
  • ● 滑動窗口 (Sliding Window)：理解數據如何「先進先出 (FIFO)」。
  • ⚠️ 延遲代價：窗口越大，數據越平滑，但對真實變化的反應就會越慢 (Lag)。

✍️ 手算挑戰

數據流：[10, 12, 11, 25, 12]。若視窗為 3，請計算最後一筆數據進入後的平均值。

moving_average.js 
function movingAverage(newData) {
  window.push(newData);
  if (window.length > N) {
    window.shift();
  }
  return sum(window) / window.length;
}

📉 3. 一階低通濾波 (Low-pass Filter) (15 min)

核心公式
Output = α × New + (1 - α) × Old

指數加權平均：不需要記憶大量歷史點，只需保留上一次的濾波結果，節省記憶體。

參數 α 的意義
  • • α 越小：系統越穩定但越遲鈍。
  • • α 越大：系統越靈敏但越抖動。

💡 預習任務：情境選擇

思考在「緊急避障」與「穩定巡航」兩種場景下，$\alpha$ 值應該如何取捨？

快速反應
巡航
極致平穩
