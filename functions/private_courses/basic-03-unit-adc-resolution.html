🌉 1. 類比與數位的橋樑：什麼是 ADC？

物理訊號：世界是類比 (Analog) 的。光線強弱、氣溫、甚至是你的聲音，它們都是平滑、連續地變動。

數位轉換：ADC (Analog-to-Digital Converter) 就像一把「數位量尺」，將 0V 到 3.3V 的電壓範圍切成好幾格，量測目前在哪一格。

解析度 (Resolution) 的差異：

  • 10-bit Arduino Uno: 切成 \(2^{10} = 1024\) 格 (0–1023)
  • 12-bit ESP32 預設: 切成 \(2^{12} = 4096\) 格 (0–4095)

🪄 2. 數學映射：Map 函式的魔力

為什麼要 Mapping？感測器讀到的是 0–4095，但馬達控制需要的是 0–255，或者我們想顯示真實電壓 0.0V–3.3V。

線性換算公式：

\[TargetValue = (SourceValue - In_{min}) \times \frac{(Out_{max} - Out_{min})}{(In_{max} - In_{min})} + Out_{min}\]

⚠️ 3. ESP32 ADC 的特性與限制

📉

非線性區 (Non-linearity)

ESP32 的 ADC 在極低電壓 (接近 0V) 與極高電壓 (接近 3.3V) 附近並不準確，會出現斜率變平的現象。

📶

ADC1 與 ADC2 的差異

當 Wi-Fi 開啟時，ADC2 會失效！ 專案中若需要網路功能，請優先選擇 ADC1 (GPIO 32-39)。

作業指引 (Assignment Guide)

作業 3-1：「數位量尺」：電壓量測實驗

  • 角色扮演： 測試工程師 (Test Engineer)。
  • 執行方式： 使用可變電阻，建立 ADC 讀值與實際電壓（由三用電表量測）的對應表。記錄並觀察 ESP32 的 0V-0.1V 與 3.2V-3.3V 的死區 (Dead-zone) 現象。
  • 評分重點 : 成功量測並記錄至少 5 個電壓點與對應的 ADC 數值。

作業 3-2：「自製感測器變量儀」

  • 角色扮演： 控制系統工程師 (Control Systems Engineer)。
  • 執行方式： 將 ADC 讀值映射為 PWM 輸出以控制 LED 亮度。實作「反向控制」（數據越大燈越暗）與簡單的「移動平均」濾波演算法。
  • 評分重點 : 程式能流暢地根據類比輸入改變 LED 亮度，且數據跳動明顯減少。

作業 3-3：「精準轉向模擬器」

  • 角色扮演： 運動控制專家 (Motion Control Specialist)。
  • 執行方式： 利用可變電阻模擬搖桿，計算左右輪速差。實作「死區設定」（中心點 ±50 範圍內不觸發），以消除靜態時的小幅跳動。
  • 評分重點 : 掌握線性映射邏輯，並能有效實作死區過濾以提升系統穩定性。

B. 參考範例代碼 (Reference Example Code)

main.cpp (ADC 讀取與映射)

#include <Arduino.h>

const int potPin = 32; // 使用 ADC1 引腳避開 Wi-Fi 衝突

void setup() {
  Serial.begin(115200);
  analogReadResolution(12); // 設定 ADC 解析度為 12-bit
  analogSetAttenuation(ADC_11db); // 設定衰減量以讀取 3.3V
}

void loop() {
  int rawValue = analogRead(potPin);
  // 映射到 0-100%
  int percent = map(rawValue, 0, 4095, 0, 100);
  Serial.printf("Raw: %d | Percent: %d%%\n", rawValue, percent);
  delay(100);
}

C. 自動評分 (Autograding) 的設計

  • 核心 API 檢索：： 偵測程式中是否呼叫了 analogRead()。
  • 邏輯邊界測試：： 透過腳本驗證學員的 map() 參數是否對齊 12-bit 解析度 (4095)。
