進階 15 - 單元 1

系統架構全覽與數據流分析

從觸控到驅動：解構無人車系統中，數據如何跨越空間賦予機械生命。

💡

為什麼要學數據流？

開發者常常迷失在代碼細節中，卻忽略了系統的全貌。理解數據從「手指觸摸螢幕」到「馬達轉動」的完整路徑，是具備「端到端 (End-to-End)」除錯能力的關鍵。

🎯

本單元重點

  • ✓掌握感知、傳輸、執行三層架構
  • ✓理解序列化 (Serialization) 與數據映射
  • ✓體會同步與非同步的設計差異

🏗️ 1. 系統三層架構全覽 (20 min)

1. 感知/控制層 (Web UI)

捕捉觸控事件，產生原始的 (x, y) 座標數據。

Event: touchstart

2. 傳輸層 (BLE / Wi-Fi)

負責數據封裝，跨越空間飛向 ESP32。

Packet: {"cmd":"MOVE"}

3. 執行/反饋層 (ESP32)

解析封包，轉化為 PWM 訊號驅動馬達。

analogWrite(PIN, 255)

🔄 2. 數據的變身術：從 String 到 PWM (20 min)

序列化 (Serialization)

必須將概念轉換為機器可讀格式。JSON 方便閱讀但體積大，Binary 難讀但傳輸快。

數值映射 (Mapping)

搖桿座標轉 PWM 的公式：pwm = map(val, 0, 100, 0, 255)

// Data Pipeline Visualization

UI -> { x: 0, y: 1.0 }

BLE -> "{\"y\":1.0}"

PWM -> DutyCycle: 255

核心實作：數據流分析儀

利用工具驗證數據完整性，並在極限環境下測試系統性能。

📊 60 min

【作業 1-1】

數據流向分析儀

利用 Console 與 Serial Monitor 進行雙向監測。

  • 前端埋點 console.log
  • ESP32 印出原始 Hex 碼
  • 核對 Byte Order 一致性
⏱️ 60 min

【作業 1-2】

數據延遲與瓶頸挑戰

手動改變傳輸頻率與大小，測試極限。

  • 100Hz 高頻壓力測試
  • 注入巨型 JSON 字串
  • 記錄延遲對即時性的影響
🛡️ 45 min

【作業 1-3】

系統異常模擬與防禦

設計具備自癒能力的系統。

  • 亂碼注入測試 Try-Parse
  • 模擬物理斷線保護機制
  • 設計對外錯誤提示 UI
