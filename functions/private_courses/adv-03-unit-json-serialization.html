進階 3 - 單元 2

JSON 序列化
與反序列化

揭密 JSON：複雜資料如何通過電線傳輸，從物件到字串的華麗轉身。

📦 1. 為什麼要序列化？ (20 min)

物理限制：理解通訊媒介（藍牙、Wi-Fi）只能傳送一連串的「位元組（Bytes）」，無法直接傳送一個「JavaScript 物件」。

📤 序列化 (Serialization)

將記憶體中的物件狀態「壓平」成文字格式（如 JSON 字串）的過程。

📥 反序列化 (Deserialization)

將接收到的文字格式「重新組裝」成可操作物件的過程。

🪑 學習任務：家具郵寄比喻

想像你要郵寄一張椅子給朋友。你必須先拆解它並裝箱（序列化），朋友收到後再按照說明書組裝（反序列化）。

📜 2. JSON 格式規範與類型 (25 min)

合法格式：理解 JSON 支援的數據類型（字串、數字、布林值、陣列、物件）。

⚠️ 常見錯誤：雙引號

✔ {"speed": 50} // 正確

✘ {speed: 50} // 錯誤：Key 必須用雙引號

🔍 預習任務

觀察一段複雜的無人車配置 JSON，嘗試識別出哪些是「數值（Number）」，哪些是「嵌套物件（Nested Object）」。

🛠️ 3. 語法工具箱 (15 min)

🌐 JavaScript 端

  • JSON.stringify(obj)
  • JSON.parse(string)

🦾 C++ (ESP32) 端

  • serializeJson(doc, output)
  • deserializeJson(doc, input)

💡 節省頻寬的技巧

如何設計精簡的 Key 值？
例如使用 {"d": 10} 代替 {"distance": 10}。

學習目標：在線上編譯器中寫出一個能產生目標 JSON 格式的代碼片段。

數據通訊實驗室

掌握物件與字串的轉換，建立穩健的 IoT 數據交換機制。

  • 建立包含 battery, is_moving, coordinates 的 JS 物件
  • 使用 JSON.stringify() 並計算字串位元組 (Bytes)

  • 實作 try...catch 邏輯來處理格式錯誤的 JSON
  • 利用物件解構賦予缺失欄位預設值
  • 嘗試解析一段虛擬 ESP32 數據並顯示於網頁上

  • 進行數值截斷（如 3.14159 -> 3.14）觀察縮減程度
  • 模擬高頻率發送時瀏覽器的記憶體佔用情況

進階課程 3 - 單元 2 | JSON 序列化與反序列化實作

作業指引 (Assignment Guide)

作業 2-1：封包工廠：狀態物件序列化

  • 角色扮演： 前端資料處理員。
  • 執行方式： 建立一個包含 `battery` (0-100)、`is_moving` (bool)、`speed` (float) 的 JS 物件，使用 `JSON.stringify()` 轉換後，計算字串的大小（位元組），並與原始物件在記憶體中的推估大小做對比。
  • 評分重點： 能否寫出結構完整、符合規範的 JSON 字串。

作業 2-2：解析挑戰：防礙型 JSON 除錯

  • 角色扮演： 軟體除錯工程師。
  • 執行方式： 故意傳入一個格式錯誤的字串（如遺失雙引號或多出逗號），撰寫 `try...catch` 捕捉錯誤，並回傳一個「預設狀態物件」以確保系統不崩潰。
  • 評分重點： 代碼是否有考慮到「防禦性編程」的概念。

作業 2-3：數據精簡賽：Payload 瘦身計劃

  • 角色扮演： 系統架構師。
  • 執行方式： 將 `{"distance": 12.34567, "status": "active"}` 精簡為縮寫鍵名與截斷小數點後的版本（如 `{"d":12.3,"s":1}`），並測量節省了多少物理頻寬。
  • 評分重點： 精簡後的 JSON 是否仍具備可讀性與唯一性。
