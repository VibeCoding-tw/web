進階 3 - 單元 2

JSON 序列化
與反序列化

揭密 JSON：複雜資料如何通過電線傳輸，從物件到字串的華麗轉身。

📦 1. 為什麼要序列化？ (20 min)

物理限制：理解通訊媒介（藍牙、Wi-Fi）只能傳送一連串的「位元組（Bytes）」，無法直接傳送一個「JavaScript 物件」。

📤 序列化 (Serialization)

將記憶體中的物件狀態「壓平」成文字格式（如 JSON 字串）的過程。

📥 反序列化 (Deserialization)

將接收到的文字格式「重新組裝」成可操作物件的過程。

🪑 學習任務：家具郵寄比喻

想像你要郵寄一張椅子給朋友。你必須先拆解它並裝箱（序列化），朋友收到後再按照說明書組裝（反序列化）。

📜 2. JSON 格式規範與類型 (25 min)

合法格式：理解 JSON 支援的數據類型（字串、數字、布林值、陣列、物件）。

⚠️ 常見錯誤：雙引號

✔ {"speed": 50} // 正確

✘ {speed: 50} // 錯誤：Key 必須用雙引號

🔍 預習任務

觀察一段複雜的無人車配置 JSON，嘗試識別出哪些是「數值（Number）」，哪些是「嵌套物件（Nested Object）」。

🛠️ 3. 語法工具箱 (15 min)

🌐 JavaScript 端

  • JSON.stringify(obj)
  • JSON.parse(string)

🦾 C++ (ESP32) 端

  • serializeJson(doc, output)
  • deserializeJson(doc, input)

💡 節省頻寬的技巧

如何設計精簡的 Key 值？
例如使用 {"d": 10} 代替 {"distance": 10}。

學習目標：在線上編譯器中寫出一個能產生目標 JSON 格式的代碼片段。

數據通訊實驗室

掌握物件與字串的轉換，建立穩健的 IoT 數據交換機制。

🏭 60 min

【作業 2-1】封包工廠

任務目標：實作一個能將無人車多樣感測器數據打包並傳送的網頁端功能。

  • 建立包含 battery, is_moving, coordinates 的 JS 物件
  • 使用 JSON.stringify() 並計算字串位元組 (Bytes)
  • UI 即時顯示「目前準備傳送的原始字串」以供除錯
🧩 60 min

【作業 2-2】解析挑戰

任務目標：實作防禦性程式碼，確保網頁不會因為收到的數據出錯而當機。

  • 實作 try...catch 邏輯來處理格式錯誤的 JSON
  • 利用物件解構賦予缺失欄位預設值

⚡ 45 min

【作業 2-3】效能優化

任務目標：在不遺失資訊的前提下，極大化縮短序列化後的字串長度。

  • 比較冗長鍵名與縮寫鍵名對頻寬的影響
  • 進行數值截斷（如 3.14159 -> 3.14）觀察縮減程度
