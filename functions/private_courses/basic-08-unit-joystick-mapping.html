基礎 8 - 控制數學

搖桿數據的本質：
歸一化、映射與死區

將混亂的物理像素轉化為精確的硬體指令，掌握操縱感的科學。

📏 1. 什麼是歸一化 (Normalization)？ (30 min)

核心概念：將不規則的數值（如螢幕上的 142px）轉化為標準的範圍（如 \(0.0\) 到 \(1.0\) 或 \(-1.0\) 到 \(1.0\)）。

🌟 為什麼重要？

不同手機的解析度不同，搖桿半徑可能是 80px 或 150px。若不歸一化，同樣的滑動距離在不同手機上會產生不同的車速。

歸一化公式

\[\text{Normalized Value} = \frac{\text{Offset}}{\text{Max Radius}}\]

🎯 線上任務

計算在 12-bit ADC 下，當讀取值為 3072 時，對應的歸一化座標是多少？（假設範圍為 0-4095）

🗺️ 2. 數值映射 (Mapping) 的邏輯 (30 min)

定義：將歸一化後的數值轉換為硬體能理解的區間。

🏎️ PWM 控制

\(-1.0\) 到 \(1.0\) 映射為 \(-255\) 到 \(255\)（馬達驅動器）。

📐 舵機控制

\(-1.0\) 到 \(1.0\) 映射為 \(0^\circ\) 到 \(180^\circ\)。

// 萬用映射函式範例 (C++)
float mapValue(float x, float inMin, float inMax, float outMin, float outMax) {
  return (x - inMin) * (outMax - outMin) / (inMax - inMin) + outMin;
}

⚙️ 學習任務

練習寫一個 map(value, inMin, inMax, outMin, outMax) 函式，並思考為什麼我們需要這個工具？

🌑 3. 死區 (Deadzone) 的設計 (20 min)

概念：搖桿中心微小的偏移不應觸發馬達，以防止硬體因雜訊而抖動。

🛡️ 為什麼需要死區？

搖桿在中央時，讀取值往往不會剛好是 0。若不設定死區，微小的雜訊會讓馬達微弱轉動，造成能量浪費或硬體抖動。

if (abs(value) < 0.1) {
  value = 0; // 忽略小於 10% 的偏移
}

🔍 學習目標

理解為何需要設定一個閥值（如 10%），低於此值一律視為 0。這在實際硬體控制中是極為重要的安全機制。

核心任務：操縱感的終極調校

從數值歸一化到非線性映射，打造職人級的控制曲線。

📉 60 min

【作業 1-1】

「數據標準化」

實作搖桿座標轉換邏輯，並在 UI 上即時顯示歸一化後的 (X, Y)。不論在什麼尺寸的手機上，輸出範圍始終固定為 [-1, 1]。

🚀 60 min

【作業 1-2】

「指數型油門」

實作 \(y = x^2\) 或 \(y = x^3\) 的控制曲線。讓無人車在低速時更易微調（溫和），高速時具備爆發力。

📦 45 min

【作業 1-3】

「數據打包」

將歸一化的數據封裝進 JSON 並限制小數點後兩位。實作「數值未改變則不發送」機制，降低通訊負擔。
