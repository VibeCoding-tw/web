基礎 4 - H 橋驅動

H 橋驅動原理：
解鎖馬達控制的鑰匙

掌握馬達正反轉的核心電路架構，實現精準的動力控制與動態煞車。

🏗️ 1. H 橋的構造：為什麼叫 H？

電路幾何： 想像一個「H」字型，中間橫桿是馬達，四個支柱是開關（通常是 MOSFET 或電晶體）。

電流路徑： 理解電流必須從電源正極出發，經過一個高側開關、穿過馬達、再經由一個對角的低側開關回到負極。

核心構造關鍵：

  • 🔹 四個開關（S1, S2, S3, S4）控制兩個迴路。
  • 🔹 對角線導通決定電流流向。

🕹️ 2. 四種核心邏輯狀態

正轉 (Forward)

對角線開關 $S_1, S_4$ 導通，通正向電壓。

反轉 (Reverse)

對角線開關 $S_2, S_3$ 導通，通反向電壓。

煞車 (Brake/Short)

底部開關同時導通，產生反向電動勢抵消。

空轉 (Coast/Idle)

所有開關切斷，馬達因慣性滑行。

⚠️ 禁忌狀態：直通短路 (Shoot-through)

絕對不能同時導通同一側的上下開關（例如 S1 與 S3），這會導致電源直接對地短路，燒毀驅動晶片！

📊 邏輯真值表 (以 TB6612 為例)

IN1 IN2 PWM (EN) 觀察結果
HIGH LOW 255 正轉 (全速)
LOW HIGH 128 反轉 (半速)
HIGH HIGH - 煞車 (Brake)
LOW LOW - 停止 (Coast)

⚡ 3. 結合 PWM 控制速度

H 橋不只是「開」或「關」。透過在對角線導通期間加上 PWM 訊號，我們可以控制馬達獲得的平均電壓。

  • ⚡

    脈衝驅動

    PWM 的占空比直接決定了推動力。50% Duty Cycle 代表一半的能量。

  • 🔧

    智慧 API 封裝

    設計 drive(int speed) 函式，正數代表 IN1=H, IN2=L, PWM=speed；負數代表 IN1=L, IN2=H, PWM=|speed|。

作業指引 (Assignment Guide)

作業 2-1：「邏輯真值表」實測

  • 角色扮演： 韌體驗證工程師 (Firmware Validation Engineer)。
  • 執行方式： 編寫程式碼透過 ESP32 腳位（AIN1, AIN2, PWM）控制馬達驅動模組。手動切換四種邏輯狀態（正轉、反轉、煞車、空轉），驗證真實電路是否符合 H 橋真值表，並截圖 Serial 紀錄。
  • 評分重點 : 能正確對應控制腳位與馬達動作，且清楚「煞車」與「空轉」在物理感受上的差異。

作業 2-2：「急煞 vs. 緩停」效能對比

  • 角色扮演： 機械控制工程師 (Mechanical Control Engineer)。
  • 執行方式： 設定馬達全速轉動後，分別測試「空轉 (Coast)」與「煞車 (Brake)」兩種停止指令，測量並記錄小車滑行距離的差異，並討論反向電動勢帶來的阻尼感。
  • 評分重點 : 數據準確且分析合理，能理解為何在無人車避障時需要使用「煞車」模式而非「停止」模式。

作業 2-3：方向與速度綜合控制

  • 角色扮演： 高級動力系統開發員 (Senior Powertrain Developer)。
  • 執行方式： 結合 PWM 與 H 橋邏輯，封裝 drive(int speed) API（參數正值前進，負值後退，0 為煞車）。撰寫一段測試程式，完成由靜止緩加速至全速，再突發急煞，最後緩慢倒車的平滑流程。
  • 評分重點 : 成功實作死區補償 (Dead Zone Compensation) 與平滑切換邏輯，控制過程無劇烈抖動。

B. 參考範例代碼 (Reference Example Code)

main.cpp (H 橋控制 API)

#include <Arduino.h>

const int IN1 = 12;
const int IN2 = 13;
const int PWM_CH = 0;

void setup() {
  ledcSetup(PWM_CH, 15000, 8);
  ledcAttachPin(14, PWM_CH);
  pinMode(IN1, OUTPUT);
  pinMode(IN2, OUTPUT);
}

void drive(int speed) {
  if (speed > 0) {
    digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
    ledcWrite(PWM_CH, speed);
  } else if (speed < 0) {
    digitalWrite(IN1, LOW); digitalWrite(IN2, HIGH);
    ledcWrite(PWM_CH, -speed);
  } else {
    digitalWrite(IN1, HIGH); digitalWrite(IN2, HIGH); // Brake
  }
}

C. 自動評分 (Autograding) 的設計

  • 核心腳位檢查：： 掃描 pinMode 是否正確配置 IN1, IN2 為輸出。
  • 邏輯分支測試：： 模擬輸入 speed = -200，驗證 IN1 是否為 Low 且 IN2 為 High。
