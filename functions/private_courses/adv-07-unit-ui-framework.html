進階 7 - 單元 1

UI 框架與
組件化設計 (Components)

解構現代網頁的核心：從 React 哲學到組件封裝，學習如何打造可重複使用且高效的介面模組。

🧱 1. 什麼是元件化設計 (Component-Based Design)？ (25 min)

  • ● 核心概念：將頁面拆成獨立、封裝且可重複使用的單元。
  • ● 封裝 (Encapsulation)：一個元件（如「馬達狀態列」）應包含自己的 HTML 結構、CSS 與邏輯，而不干擾其他部分。

🧩 學習任務

觀察 Tesla 螢幕或遊戲介面，嘗試用紙筆劃出其中的「原子型元件」與「組合型元件」。

🔌 2. 資料傳遞：Props 與 Events (25 min)

單向資料流

  • 從上到下 (Props)： 父元件將「藍牙連線狀態」傳遞給子元件。
  • 從下到上 (Events)： 使用者操作搖桿時，子元件通知父層發送 BLE 指令。

🔍 學習任務：Bug 追蹤

理解「單向資料流」的優點——當 UI 出現 Bug 時，更容易追蹤數據是從哪裡出錯的。

⚡ 3. UI 框架的價值：Reactive (20 min)

響應式更新：當 ESP32 回傳電量為 10% 時，UI 會「自動」更新顯示，而不需要手動操作 DOM。

🧠 預習任務

思考當控制面板元件越來越多時，如果沒有框架管理，代碼會如何變得混亂？

// 傳統 DOM 操作 vs. 框架
document.getElementById('battery').innerText = '10%';
// 響應式
state.battery = 10; // UI 自動更新

🖥️ 4. 佈局架構與遙測視覺化

內容： 介紹如何使用 CSS Grid 與 Flexbox 快速搭建響應式儀表板。

重點： 認識指針式 (Gauge)、長條圖 (Bar) 與即時折線圖 (Line Chart) 的適用場景。

🎯 學習目標

在電腦上先完成一個不含真實數據的「靜態遙測介面」。

核心實驗室：元件化 UI 開發

採用 Git Workflow：領取 GitHub Classroom 任務，實作過濾算法並提交 PR。

📊 60 min

【作業 1-1】原子元件

任務內容：設計一個接收 label, value, color 三個 Props 的通用數值顯示元件。

  • 屬性設定：支援自定義名稱與顏色
  • 動態渲染：電量 < 20% 時背景自動變紅
  • 目標：學會建立可多次調用但表現不同的基礎元件
🕹️ 60 min

【作業 1-2】搖桿元件

任務內容：將 Canvas 搖桿邏輯封裝，計算 (x, y) 並發出 joystick-move 事件。

  • 解耦邏輯：元件只負責計算，不處理 BLE
  • 自定義事件：Emit 座標數據給父層
  • 分工原則：視覺歸元件，通訊歸父層
📐 45 min

【作業 1-3】佈局優化

任務內容：利用 CSS Grid 設計適應手機（縱向）與電腦（橫向）的儀表板佈局。

  • 響應式配置：自動切換單欄與雙欄佈局
  • 狀態同步：點擊連線後，全局進入 active 模式
  • 目標：提升控制介面在不同裝置上的專業體驗

作業指引 (Assignment Guide)

進階 7 - UI 框架與元件化

3-1 原子化標籤組件 (Atom Component)

角色扮演： 前端 UI 設計師。

執行步驟：

  1. 設計一個名為 StatusBadge 的 JS 函式。
  2. 接收一個 type 參數（如 online, offline）。
  3. 根據 type 返回帶有不同 CSS 樣式的 HTML 字串片段（如綠色或紅色提示燈）。

評分重點： 函式是否能根據參數正確渲染對應的樣式，展現「樣式封裝」的思維。

3-2 數據卡片工廠 (Card Factory)

角色扮演： 介面模組化專家。

執行步驟：

  • 實作一個 DeviceCard 元件。
  • 該元件包含裝置名稱、即時數據、以及一個控制按鈕。
  • 將按鈕的 onclick 事件也封裝在元件生成過程中。

評分重點： 元件是否具備完整的「自包含力 (Self-contained)」，即 HTML、樣式與基本互動邏輯是否封裝在一起。

3-3 佈局重構挑戰

角色扮演： 系統架構師。

執行步驟：

  • 將原本雜亂的 `index.html` 中的資料欄位改寫。
  • 完全替換為由 JS 動態生成的「元件列」。
  • 確保在新增新感測器時，只需要更動數據陣列，而不需更動 HTML 結構。

評分重點： 頁面結構是否變得簡潔，並體會到「關注點分離 (Separation of Concerns)」在大型專案中的重要性。

講師指引 (Instructor Guide)

GitHub Classroom & Flipped Teaching

🚀 在 GitHub Classroom 的實戰部署

A. 範本倉庫 (Template Repo) 結構 

.
├── components/
│   ├── badge.js      # 原子化組件
│   └── card.js       # 裝置卡片組件

├── tests/
│   └── dom.test.js   # 測試：驗證 DOM 結構生成
└── README.md          # 元件 API 文檔紀錄

B. 自動評估 (Autograding) 的設計

🧱

元件純粹性測試： 模擬注入不同的 Props（屬性），檢查生成出的 HTML 字串是否符合預期的數據對應。

⚡

互動事件綁定測試： 觸發模擬點擊，驗證封裝在元件內部的 onclick 處理程序是否被正確執行。

🛠️ GitHub Classroom 詳細操作指南

STEP 1

準備範本專案

提供空元件資料夾與 JSDoc 格式定義的介面說明，引導學員建立 Template repository。

STEP 2

建立 Classroom 作業

設定「UI 一致性」驗證。著重於將「數據處理」與「畫面渲染」拆分為不同檔案。

STEP 3

學生端作業 (Flipped)

領取作業，對比重構前後的程式碼行數。提交包含「元件複用性」說明的 PR。執行 Git Push。

STEP 4

教師代碼審閱 (Review)

審核元件是否具備「高內聚低耦合」的特徵，回饋針對不同屏幕尺寸的適配建議。
