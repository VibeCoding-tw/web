進階 7 - 單元 1

現代介面架構：
UI 框架與元件化設計

想像 UI 是「樂高積木」。學習如何將複雜的控制介面拆解為獨立、封裝且可重複使用的元件。

🧱 1. 什麼是元件化設計 (Component-Based Design)？ (25 min)

  • ● 核心概念：將頁面拆成獨立、封裝且可重複使用的單元。
  • ● 封裝 (Encapsulation)：一個元件（如「馬達狀態列」）應包含自己的 HTML 結構、CSS 與邏輯，而不干擾其他部分。

🧩 學習任務

觀察 Tesla 螢幕或遊戲介面，嘗試用紙筆劃出其中的「原子型元件」與「組合型元件」。

🔌 2. 資料傳遞：Props 與 Events (25 min)

單向資料流

  • 從上到下 (Props)： 父元件將「藍牙連線狀態」傳遞給子元件。
  • 從下到上 (Events)： 使用者操作搖桿時，子元件通知父層發送 BLE 指令。

🔍 學習任務：Bug 追蹤

理解「單向資料流」的優點——當 UI 出現 Bug 時，更容易追蹤數據是從哪裡出錯的。

⚡ 3. UI 框架的價值：Reactive (20 min)

響應式更新：當 ESP32 回傳電量為 10% 時，UI 會「自動」更新顯示，而不需要手動操作 DOM。

🧠 預習任務

思考當控制面板元件越來越多時，如果沒有框架管理，代碼會如何變得混亂？

// 傳統 DOM 操作 vs. 框架
document.getElementById('battery').innerText = '10%';
// 響應式
state.battery = 10; // UI 自動更新

🖥️ 4. 佈局架構與遙測視覺化

內容： 介紹如何使用 CSS Grid 與 Flexbox 快速搭建響應式儀表板。

重點： 認識指針式 (Gauge)、長條圖 (Bar) 與即時折線圖 (Line Chart) 的適用場景。

🎯 學習目標

在電腦上先完成一個不含真實數據的「靜態遙測介面」。

核心實驗室：元件化開發實戰

從原子元件到複雜佈局，打造專業級的控制中心。

📊 60 min

【作業 1-1】原子元件

任務內容：設計一個接收 label, value, color 三個 Props 的通用數值顯示元件。

  • 屬性設定：支援自定義名稱與顏色
  • 動態渲染：電量 < 20% 時背景自動變紅
  • 目標：學會建立可多次調用但表現不同的基礎元件
🕹️ 60 min

【作業 1-2】搖桿元件

任務內容：將 Canvas 搖桿邏輯封裝，計算 (x, y) 並發出 joystick-move 事件。

  • 解耦邏輯：元件只負責計算，不處理 BLE
  • 自定義事件：Emit 座標數據給父層
  • 分工原則：視覺歸元件，通訊歸父層
📐 45 min

【作業 1-3】佈局優化

任務內容：利用 CSS Grid 設計適應手機（縱向）與電腦（橫向）的儀表板佈局。

  • 響應式配置：自動切換單欄與雙欄佈局
  • 狀態同步：點擊連線後，全局進入 active 模式
  • 目標：提升控制介面在不同裝置上的專業體驗
