視覺追蹤控制策略：
比例控制 (P-Control)

賦予機器人「直覺」：學習如何利用數學公式讓無人車根據誤差大小，聰明地調整轉向力道。

🧠 1. 什麼是比例控制 (P-Control)？ (20 min)

  • ● 核心哲學：修正的力道與誤差成正比。「誤差越大，修正越猛；誤差越小，修正越溫柔」。
  • ● 數學公式：
    Output = Kp × Error
  • • Output：馬達的修正量（或轉向角度）。
  • • Error：物體中心與畫面中心的偏差。
  • • Kp：比例增益（Proportional Gain），這是系統的「個性」。

🎛️ 2. 比例增益 Kp 的行為特徵 (30 min)

  • S
    小 Kp：反應遲鈍，修正緩慢，車子可能會衝出跑道（追蹤不上）。
  • B
    大 Kp：反應靈敏，但過度修正會導致系統產生「震盪 (Oscillation)」，車子會像喝醉一樣左右蛇行。

觀看 P 控制器的動態模擬影片，觀察在不同 $Kp$ 值下，系統趨近目標線的曲線圖（是平滑接近，還是反覆橫跳？）。

📉 3. 穩態誤差 (Steady-State Error) 的預習 (20 min)

P 控制的原生缺陷：

當誤差非常小時，輸出也會趨近於零。如果環境存在阻力（如地面摩擦力），微小的輸出力道可能推不動馬達，這導致系統永遠無法完全精確地重合目標，這就是「穩態誤差」。

💡 思考題

如果路面摩擦力很大，微小的 Output 推不動馬達，P 控制該如何克服？我們是否需要一種能「累積誤差」的力量？（提示：這將是 I - 積分控制的登場時刻）。

作業指引 (Assignment Guide)

作業 3-1：實作 P-Controller 核心函式

  • 角色扮演： 控制系統工程師。
  • 執行方式： 在網頁端實作 `calculatePControl(error, Kp)` 函式。將誤差乘上比例增益，並加上一個「限幅 (Saturation)」邏輯，確保輸出不超過馬達的極限值。
  • 評分重點： 代碼邏輯是否簡潔且具備防止「輸出溢位 (Overflow)」的保護機制。

作業 3-2：「Kp 調音師」：動態參數調校

  • 角色扮演： 現場與調試工程師 (Field Engineer)。
  • 執行方式： 透過控制介面的 Kp 滑桿，觀察無人車在跟隨目標時的表現。紀錄三個關鍵 Kp 值：剛好動不了、完美追隨、開始瘋狂震盪 (Over-shooting)。
  • 評分重點： 能否在 GitHub README 中清楚分析不同 $Kp$ 值對系統「穩定性 (Stability)」與「反應速度 (Responsiveness)」的影響。

作業 3-3：追蹤壓力測試（速度 vs. 誤差）

  • 角色扮演： 系統魯棒性測試員。
  • 執行方式： 在維持 $Kp$ 的情況下，逐漸提升車子的「基礎前行速度」。觀察在高速彎道中，單純的 P 控制是否還能將誤差維持在容許範圍內。
  • 評分重點： 發現並描述 P 控制在處理高速慣性時的極限（例如追不到彎或是嚴重蛇行），作為後續引進 D (微分) 控制的動機。
