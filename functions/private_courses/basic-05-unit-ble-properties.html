基礎 5 - 藍牙通訊進階

BLE 核心機制：
讀取、寫入與通知

掌握 GATT 特徵值屬性，設計高效且省電的物聯網通訊地圖。

📖 1. 讀取 (Read)：當你需要「查詢」時 (15 min)

定義：客戶端（手機）主動向伺服器（車子）要求索取數據。

運作邏輯：就像查字典

只有在手機發問時，ESP32 才會回傳數據。這是最被動的通訊方式。

適合場景

韌體版本、設備 ID、PID 靜態參數。

學習任務

思考：為什麼我們「不常用」Read 來取得即時超音波距離？

ESP32 實作預覽

pChar->setValue("V1.0.0");
// 手機發起 Read 時
// ESP32 回傳目前 setValue 的值

✍️ 2. 寫入 (Write)：當你需要「下令」時 (20 min)

手機主動傳送資料給 ESP32，通常用於控制指令。

🛡️ Write with Response

安全模式。ESP32 收到後會回傳确认（ACK），保證指令傳達。適用於關鍵設定。

⚡ Write Without Response (Command)

快速模式。手機只管丟指令，不管車子有沒有收到。適用於高頻率搖桿控制。

適合場景：馬達開關、目標速度、顏色切換、搖桿座標。

🏗️ Callbacks 實作邏輯

當寫入發生時，ESP32 會觸發一個「回呼函式」：

class MyCallbacks: public BLECharacteristicCallbacks {
  void onWrite(BLECharacteristic *pChar) {
    std::string value = pChar->getValue();
    if (value == "1") digitalWrite(LED, HIGH);
  }
};

🔔 3. 通知 (Notify)：當你需要「即時同步」時 (25 min)

定義：伺服器（車子）主動推播資料給手機。

運作邏輯：訂閱機制

手機先「訂閱」該特徵值，之後只要 ESP32 數據一變動，車子就會立刻主動推送。

🔋 習題 POWER SAVING

比較 Read 與 Notify，哪一個對手機的省電效果更好？（提示：減少發問次數）。

即時轉速、循跡誤差值、電池電量警報。
