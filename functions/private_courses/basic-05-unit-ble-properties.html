基礎 5 - 藍牙通訊進階

BLE 核心機制：
讀取、寫入與通知

掌握 GATT 特徵值屬性，設計高效且省電的物聯網通訊地圖。

📖 1. 讀取 (Read)：當你需要「查詢」時 (15 min)

定義：客戶端（手機）主動向伺服器（車子）要求索取數據。

運作邏輯：就像查字典

只有在手機發問時，ESP32 才會回傳數據。這是最被動的通訊方式。

適合場景

韌體版本、設備 ID、PID 靜態參數。

學習任務

思考：為什麼我們「不常用」Read 來取得即時超音波距離？

ESP32 實作預覽

// ESP32 回傳目前 setValue 的值

✍️ 2. 寫入 (Write)：當你需要「下令」時 (20 min)

手機主動傳送資料給 ESP32，通常用於控制指令。

🛡️ Write with Response

安全模式。ESP32 收到後會回傳确认（ACK），保證指令傳達。適用於關鍵設定。

⚡ Write Without Response (Command)

快速模式。手機只管丟指令，不管車子有沒有收到。適用於高頻率搖桿控制。

適合場景：馬達開關、目標速度、顏色切換、搖桿座標。

🏗️ Callbacks 實作邏輯

當寫入發生時，ESP32 會觸發一個「回呼函式」：

class MyCallbacks: public BLECharacteristicCallbacks {
  void onWrite(BLECharacteristic *pChar) {
    std::string value = pChar->getValue();
    if (value == "1") digitalWrite(LED, HIGH);
  }
};

🔔 3. 通知 (Notify)：當你需要「即時同步」時 (25 min)

定義：伺服器（車子）主動推播資料給手機。

運作邏輯：訂閱機制

手機先「訂閱」該特徵值，之後只要 ESP32 數據一變動，車子就會立刻主動推送。

作業指引 (Assignment Guide)

【作業 3-1】「儀表板實作」：混合屬性配置

  • 角色扮演： 嵌入式 UI 工程師 (Embedded UI Engineer)。
  • 執行方式： 在同一個 BLE Service 下配置多個特徵值：一個可讀取學員學號，一個可寫入控制板載 LED，一個每秒推送隨機模擬感測數據。使用手機端 App 進行全面測試。
  • 評分重點 : 成功在一個服務中管理多種屬性，且各功能與預期一致且無干擾。

【作業 3-2】「指令延遲大挑戰」：Write vs NoResp

  • 角色扮演： 通訊效能分析師 (Communication Performance Analyst)。
  • 執行方式： 每秒發送 50 次指令，比較 PROPERTY_WRITE 與 PROPERTY_WRITE_NR 在高頻控制下的表現，記錄是否會產生指令堆疊與延遲（Lag）。
  • 評分重點 : 數據記錄詳實，能清晰描述兩種寫入模式在不同應用場景下的優缺點。

【作業 3-3】「智慧推播：條件式 Notify」

  • 角色扮演： 低功耗系統設計師 (Low Power System Designer)。
  • 執行方式： 實作「有意義才通知」邏輯：只有當感測器數值變動超過 5%-10% 時，ESP32 才觸發 notify()。觀察並記錄在靜止與劇烈變動情況下的通訊封包密度差異。
  • 評分重點 : 代碼邏輯正確且能顯著減少不必要的數據推播，展現對省電設計的理解。

GitHub Classroom 詳細操作指南

  1. 在 GitHub 建立新 Repo（名為 basic-05-ble-prop-template）。

第二步：建立 Classroom 作業

  1. 進入 GitHub Classroom 建立「Individual Assignment」。
  2. 連結上述的範本倉庫，並勾選 "Enable feedback pull requests"。

第三步：學生端操作 (Flipped Workflow)

  1. 學生領取作業，獲得個人專屬 Repo。
  2. 配置各式屬性權限並實作對應的回呼邏輯，透過手機 App 進行互動測試。
  3. 本地開發後 Push 代碼 提交進度。

第四步：教師回饋 (Review)

  1. 進入 Feedback PR 的 Files Changed。
  2. 針對學生的「通訊封裝效率（Byte Array 處理）」與「條件通知設計 (Conditional Notify)」予以回饋。

B. 參考範例代碼 (Reference Example Code)

main.cpp (條件式 Notify)

#include <BLEDevice.h>

int lastVal = 0;
// ... 初始化代碼 ...

void loop() {
  int currentVal = analogRead(32);
  if (abs(currentVal - lastVal) > 100) { // 變動超過阈值
    pCharacteristic->setValue(currentVal);
    pCharacteristic->notify();
    lastVal = currentVal;
    Serial.println("Notified!");
  }
  delay(10);
}

C. 自動評分 (Autograding) 的設計

  • 屬性標記檢核：： 透過測試代碼自動讀取特徵值屬性，驗證是否正確包含 PROPERTY_WRITE 或 PROPERTY_NOTIFY。
  • 邏輯流程驗證：： 模擬數值小幅變動，驗證是否正確「不觸發」推送，以節省能耗。
