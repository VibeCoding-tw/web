🗺️ 1. 認識引腳地圖 (Pinout Map)

ESP32 就像瑞士軍刀，每一條腿都有多重身分。

GPIO (General Purpose)

通用型輸入輸出，你的數位開關。

ADC & DAC

類比訊號的入口(ADC)與出口(DAC)。

Touch (觸控)

內建電容式觸控功能，不用按鈕也能感應。

硬體通訊

UART (通訊)、I2C (感測器)、SPI (螢幕)。

💣 2. 絕對不能忽視的「地雷引腳」

🚫

僅限輸入 (Input Only)

GPIO 34, 35, 36, 39。它們沒有內部拉高電阻，且不能輸出訊號 (不能點亮 LED 或控制繼電器)。

⚙️

啟動配置 (Strapping Pins)

GPIO 0, 2, 5, 12, 15。這些腳位在開機瞬間決定了進入什麼模式。

⚠️

案例分析：車子為什麼不動？

「如果在 GPIO 0 接了一個強力上拉按鈕，車子可能永遠無法正常啟動，因為它會被強制拉入燒錄模式。」

生存守則

優先使用 GPIO 12 以後的通用腳位，把地雷區留給最無害的設備。

⚡ 3. 供電腳位與極限

Vin

Vin (5V)

外部 5V 輸入口。當你不用 USB 供電時，這是你的能量入口。

3V3

3V3

降壓後的穩定輸出。可以用來驅動小型的感測器。

作業指引 (Assignment Guide)

作業 3-1：「數位探針」：驗證引腳限制

  • 角色扮演： 硬體測試工程師 (Hardware Test Engineer)。
  • 執行方式： 挑戰 Input Only 與 Strapping 引腳。實測將 GPIO 34 設為輸出是否有效？將 GPIO 12 接地重啟會發生什麼事？並記錄實驗結果。
  • 評分重點 : 能正確識別哪些引腳具備輸出功能，並解釋 Strapping 引腳對開機模式的影響。

作業 3-2：「無人車引腳資源大規劃」

  • 角色扮演： 系統架構師 (System Architect)。
  • 執行方式： 針對馬達 (4 PWM)、編碼器 (2 中斷)、I2C 感測器，在 README.md 中繪製出最佳配置圖。避開地雷引腳，確保所有組件不衝突。
  • 評分重點 : 規劃的引腳分配合理，無功能衝突（如避開了 Wi-Fi 開啟時失效的 ADC2）。

作業 3-3：「類比掃描儀」：ADC 精度與底噪

  • 角色扮演： 訊號處理專家 (Signal Processing Specialist)。
  • 執行方式： 測試類比引腳讀取表現。觀察 12-bit 解析度下的數值跳動，並撰寫一段「平均濾波」程式碼來優化數據。
  • 評分重點 : 成功實作數據平滑邏輯，且讀值穩定性顯著提升。

B. 參考範例代碼 (Reference Example Code)

main.cpp (引腳定義與安全檢查)

#include <Arduino.h>

// 理解為什麼 GPIO 34-39 只有 Input 功能
#define SENSOR_PIN 34
#define LED_PIN    2  // GPIO 2 在某些板子上與 Strapping 相關

void setup() {
  Serial.begin(115200);
  pinMode(SENSOR_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  int state = digitalRead(SENSOR_PIN);
  digitalWrite(LED_PIN, state);
}

C. 自動評分 (Autograding) 的設計

  • 語法規則檢測：： 檢查程式中是否嘗試對 Input-only 引腳（如 GPIO 36）執行 pinMode(..., OUTPUT)。
  • 衝突風險檢查：： 掃描引腳宏定義，驗證是否誤用了會影響燒錄的 GPIO 引腳。
