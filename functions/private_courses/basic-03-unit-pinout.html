🗺️ 1. 認識引腳地圖 (Pinout Map)

ESP32 就像瑞士軍刀，每一條腿都有多重身分。

GPIO (General Purpose)

通用型輸入輸出，你的數位開關。

ADC & DAC

類比訊號的入口(ADC)與出口(DAC)。

Touch (觸控)

內建電容式觸控功能，不用按鈕也能感應。

硬體通訊

UART (通訊)、I2C (感測器)、SPI (螢幕)。

💣 2. 絕對不能忽視的「地雷引腳」

🚫

僅限輸入 (Input Only)

GPIO 34, 35, 36, 39。它們沒有內部拉高電阻，且不能輸出訊號 (不能點亮 LED 或控制繼電器)。

⚙️

啟動配置 (Strapping Pins)

GPIO 0, 2, 5, 12, 15。這些腳位在開機瞬間決定了進入什麼模式。

⚠️

案例分析：車子為什麼不動？

「如果在 GPIO 0 接了一個強力上拉按鈕，車子可能永遠無法正常啟動，因為它會被強制拉入燒錄模式。」

生存守則

優先使用 GPIO 12 以後的通用腳位，把地雷區留給最無害的設備。

⚡ 3. 供電腳位與極限

Vin

Vin (5V)

外部 5V 輸入口。當你不用 USB 供電時，這是你的能量入口。

3V3

3V3

降壓後的穩定輸出。可以用來驅動小型的感測器。

講師手冊 (Instructor Guide)

1. 教學重點 (Teaching Points)

  • 強調 ESP32 的引腳多樣性：並非所有 GPIO 都是一樣的。
  • 解釋 Strapping Pins (GPIO 0, 2, 5, 12, 15) 對啟動模式 (Boot Mode) 的決定性影響。
  • 討論何謂「類比引腳」與「觸摸引腳」的分配優先級。

2. 常見問題 (Common Issues)

  • 無法燒錄：通常是 GPIO 0 被拉高或 GPIO 12 被拉錯電位。
  • 程式會莫名重啟：檢查是否在 GPIO 15 上接了會造成電位波動的負載。
  • 輸出沒反應：學生常誤用 GPIO 34, 35, 36, 39 作為輸出引腳。

3. 評分標準 (Grading Criteria)

  • 基礎分：能正確列出引腳資源表並解釋其功能限制。
  • 進階分：能說明 ESP32 內部 Mux (多工器) 如何動態分配功能。
