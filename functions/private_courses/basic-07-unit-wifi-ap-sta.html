基礎 7 - 網路通訊架構

Wi-Fi 雙模式解析：
AP (自立門戶) vs. STA (加入大家庭)

掌握 ESP32 的通訊雙重身分，打造能在任何環境下連線的產品級智慧系統。

🗼 1. AP (Access Point) 模式：自立門戶 (20 min)

核心概念：ESP32 自己當「基地台」。就像車子背著一台小迷你路由器，手機可以直接連到車子發出的 Wi-Fi。

💪 優點

  • • 不依賴外部網路。
  • • 去到哪裡都能連線（適合戶外賽道）。
  • • 設備間可形成獨立的私有網段。

⚠️ 缺點

  • • 手機連上後通常無法上網。
  • • 連線距離受限於 ESP32 發射功率。
  • • 同時連進來的人數有物理限制。

🧐 學習任務

查閱 SSID 與 Password 的設定規範。思考：為什麼 AP 模式通常會有「最大連線人數」限制？這與 ESP32 的記憶體 (RAM) 有什麼關係？

🏠 2. STA (Station) 模式：加入大家庭 (20 min)

核心概念：ESP32 像手機一樣，「加入」現有的路由器（Router）。

💪 優點

  • • 可透過路由器進行長距離遠端控制。
  • • 車子可同時連網（下載更新或傳數據）。
  • • 穩定的網路基礎設施。

⚠️ 缺點

  • • 需要環境中有路由器。
  • • 路由器負載太重時延遲會變高。
  • • 依賴密碼正確與路由器開啟。

⚙️ 學習任務：DHCP 協定

為什麼車子加入網路後，IP 地址會變動？理解如何設定「固定 IP (Static IP)」，這就像幫無人車在社區裡租一個固定的停車位，確保控制端永遠找得到它。

⚡ 3. AP + STA (Dual Mode)：最強混合體 (20 min)

核心概念：同時開啟兩種模式，讓設備具備高度的彈性。

🛒 經典應用場景：配網流程 (Provisioning)

你買了一台新的智慧插座或掃地機器人，它是如何學會你家 Wi-Fi 密碼的？

  1. 插座啟動 AP 模式，發出名稱為 "Smart_XXX" 的 Wi-Fi。
  2. 你的手機連線到這個 AP，進入設定網頁。
  3. 你在手機上輸入你家的 Wi-Fi SSID 與密碼。
  4. 插座收到後，啟動 STA 模式 把身分切換到你家路由器。

講師手冊 (Instructor Guide)

教學重點 (Teaching Points)

  • 區分 AP (基地台) 與 STA (工作站) 的物理意義。
  • 解釋 WiFi 事件（Event System）：為什麼用事件監聽比在 loop 裡面輪詢連線狀態更好。
  • 討論 Provisioning (配網)：如何利用 AP 模式設定 STA 模式所需要的 SSID/Password。

學生常見問題 (Common Issues)

  • SSID 或密碼輸入錯誤，導致一直在 WIFI_REASON_AUTH_FAIL 狀態。
  • 忘記呼叫 WiFi.mode() 導致模式不如預期。
  • 在 STA 模式下沒連上網就進行 HTTP 請求，導致系統崩潰。

評分標準 (Grading)

  • 優 (90-100): 能穩定在不同模式間切換，並具備良好的重連與錯誤偵測邏輯。
  • 可 (70-89): 基本連線功能正常，但重連邏輯不夠健全（如：路由器重啟後無法自動連回）。
  • 待加強 (Below 70): 無法成功建立連線，或不理解 AP/STA 的根本差異。
