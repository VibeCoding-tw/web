高效處理感測器數據：掌握 JSON 格式協議與 JavaScript ES6 解構語法。

💎 1. 為什麼選擇 JSON？ (20 min)

跨平台通用性：JSON 是純文字格式，無論是 ESP32（C++）還是網頁（JavaScript）都能理解。

結構化數據：理解如何用一個封包同時傳遞「電量」、「速度」與「座標」多組資訊，避免混亂的逗號分隔字串。

🔍 學習任務：優勢對比

對比「逗號分隔字串 (CSV)」與「JSON」，找出 JSON 在擴充新功能（例如增加感測器）時的優勢。

esp32_payload.json 
{
  "bat": 12.1,
  "temp": 38,
  "rpm": [120, 118]
}

🎨 2. 物件解構 (Destructuring) 的藝術 (25 min)

簡化代碼：傳統寫法需要多行才能提取資料，解構語法只需一行：
const { speed, battery } = data;

預設值設定：學習在解構時加入預設值，防止當 ESP32 漏傳某個欄位時導致網頁崩潰。

🎯 學習任務

練習從深層巢狀物件（如：data.motor.left.speed）中精準提取特定數值。

// 現代解構寫法 (ES6+)

const { speed = 0, bat } = payload;

// 巢狀提取

const { motor: { left } } = data;

💡 預習任務：為什麼不能直接傳送 JS Object？

提示：傳輸媒介（電線、電磁波）只能傳送「位元組流 (Byte Stream)」。想像你要把一張完整的椅子塞進水管裡——你必須先把它拆散成木屑 (序列化)，到水管另一端再重新拼好 (反序列化)。

作業指引 (Assignment Guide)

進階 7 - JSON 與數據處理

1-1 JSON 資料提取與防錯

角色扮演： 後端數據工程師。

執行步驟：

  1. 使用一個包含感測器讀數的 JSON 字串。
  2. 實作 try...catch 區塊來處理 JSON.parse() 可能拋出的語法錯誤。
  3. 正確從解析後的物件中提取特定數值並顯示在控制台。

評分重點： 程式碼是否包含完整的異常處理 (Exception Handling) 流程。

1-2 動態 UI 生成挑戰

角色扮演： 前端介面開發者。

執行步驟：

  • 將多個 JSON 資料物件組合成一個陣列。
  • 利用 forEach 遍歷陣列。
  • 為每個物件動態生成一個 HTML 的資料卡片 (Card)。

評分重點： 是否掌握了「數據驅動 UI (Data-driven UI)」的核心概念，並能正確使用模板字串。

1-3 複雜協議格式化

角色扮演： 通訊協定設計師。

執行步驟：

  • 設計一個深層嵌套 (Nested) 的 JSON 結構，包含裝置名稱、版本、以及多個通道的參數。
  • 實作一個格式化函式。
  • 透過 JSON.stringify(obj, null, 2) 輸出具備縮排的可讀報告。

評分重點： 對於 JSON 結構美化 (Prettify) 參數的理解及運用。
