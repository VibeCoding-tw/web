💎 1. 為什麼選擇 JSON？ (20 min)

跨平台通用性：JSON 是純文字格式，無論是 ESP32（C++）還是網頁（JavaScript）都能理解。

結構化數據：理解如何用一個封包同時傳遞「電量」、「速度」與「座標」多組資訊，避免混亂的逗號分隔字串。

🔍 學習任務：優勢對比

對比「逗號分隔字串 (CSV)」與「JSON」，找出 JSON 在擴充新功能（例如增加感測器）時的優勢。

esp32_payload.json 
{
  "bat": 12.1,
  "temp": 38,
  "rpm": [120, 118]
}

🎨 2. 物件解構 (Destructuring) 的藝術 (25 min)

簡化代碼：傳統寫法需要多行才能提取資料，解構語法只需一行：
const { speed, battery } = data;

預設值設定：學習在解構時加入預設值，防止當 ESP32 漏傳某個欄位時導致網頁崩潰。

🎯 學習任務

練習從深層巢狀物件（如：data.motor.left.speed）中精準提取特定數值。

// 現代解構寫法 (ES6+)

const { speed = 0, bat } = payload;

// 巢狀提取

const { motor: { left } } = data;

💡 預習任務：為什麼不能直接傳送 JS Object？

提示：傳輸媒介（電線、電磁波）只能傳送「位元組流 (Byte Stream)」。想像你要把一張完整的椅子塞進水管裡——你必須先把它拆散成木屑 (序列化)，到水管另一端再重新拼好 (反序列化)。

作業指引 (Assignment Guide)

作業 3-1：「封包拆解員」：動態資訊看板

  • 角色扮演： 系統整合工程師。
  • 執行方式： 模擬從 ESP32 每秒接收一個 JSON 字串（包含 `bat`, `temp`, `rpm`）。實作一個解析函式，使用 ES6 物件解構語法提取這些數值。
  • 評分重點： 代碼中是否使用了 `try...catch` 來處理無效的 JSON 字串，並能將解構出的數據即時顯示在網頁 UI 上。

作業 3-2：多階層配置 (Nested Config) 管理器

  • 角色扮演： 後端通訊專家。
  • 執行方式： 建立一個複雜的巢狀組態物件（例如：`config: { motor: { kp: 1.2, speed: 100 } }`）。練習如何精準解構出深層數據，並將其修改後重新透過 `JSON.stringify()` 轉為字串。
  • 評分重點： 對於深層物件解構語法的正確應用，以及序列化後的字串格式是否合法。

作業 3-3：數據壓縮挑戰：精簡 JSON

  • 角色扮演： 性能優化工程師。
  • 執行方式： 在不影響功能的前提下，嘗試縮短 JSON 的金鑰長度（例如把 `"motorSpeed"` 改為 `"s"`），並移除不必要的空白。
  • 評分重點： 測量並比較壓縮前後的封包大小（Byte），體會大規模通訊中節約頻寬的重要性。
