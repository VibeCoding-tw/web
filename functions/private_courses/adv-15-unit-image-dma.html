影像處理與 DMA 觀念

賦予 ESP32-S3 雙眼，利用 DMA 引擎驅動高速影像數據流，實現零延遲的邊緣視覺。

為什麼需要 DMA？

在高速影像傳輸中，CPU 不應浪費在搬運數據上。DMA (Direct Memory Access) 就像專業的搬運助手，自動完成數據流動，釋放處理核心的算力。

🚀 1. 揭祕 DMA：高速數據流 (30 min)

核心概念：搬運助手的藝術

想像 CPU 是「主廚」，搬運食材（數據傳輸）如果是主廚親自做，就沒時間做菜。DMA 就是搬運工，讓數據在記憶體與鏡頭間自動流動。

輪詢 (Polling)

CPU 不斷詢問「數據好了嗎？」，搬運時 CPU 無法做其他事。

DMA 模式

硬體自動搬運，完成後才發中斷通知 CPU。效率極大化。

JPEG 解救任務

S3 搭載硬體 JPEG 編碼器，壓縮率可達 10:1 至 20:1，是網路串流的首選。

  • 🔹 RGB565：適合循線等局部運算
  • 🔹 JPEG：串流王者，具硬體加速

📼 3. LCD_CAM 與 PSRAM (20 min)

外部緩衝的角色

SRAM 只有 512KB，裝不下 720P 影像。DMA 將攝像頭數據直接傾倒進 PSRAM 中，過程完全自動化。

Grab Mode: CAMERA_GRAB_LATEST 確保每次取得都是最新影幀，降低處理延遲。

// S3 CAM Buffer Config

config.fb_count = 2;

config.fb_location = CAMERA_FB_IN_PSRAM;

config.grab_mode = CAMERA_GRAB_LATEST;

作業指引 (Assignment Guide)

作業 3-1：「零延遲」影像捕獲測試

  • 角色扮演： 嵌入式影像工程師。
  • 執行方式： 設定 ESP32-S3 的 LCD_CAM 介面，開啟 DMA 傳輸，並在網頁端測量從「快門觸發」到「首個 Byte 到達」的時間。
  • 評分重點： 能否成功將影像直接搬運至 PSRAM 而不佔用 CPU 週期，並證明影像數據的完整性（無斷層）。

作業 3-2：影像數據流分析儀

  • 角色扮演： 訊號處理專家。
  • 執行方式： 分析傳輸過程中的 JPEG 標頭檔，計算單張圖片的平均壓縮率與傳輸帶寬佔用。
  • 評分重點： 提交一份「影像質量 vs 帶寬」的對照表，找出在維持辨識穩定度下的最低品質係數 (Quality Factor)。

作業 3-3：雙緩衝區 (Double Buffering) 優化實作

  • 角色扮演： 驅動程式開發人員。
  • 執行方式： 修改底層驅動，實作輪替 Buffer 機制，讓 DMA 在傳輸 Buffer A 的同時，CPU 正在處理 Buffer B。
  • 評分重點： 對比優化前後的 Frame Rate (FPS)，驗證是否存在影像撕裂 (Tearing) 現象。
