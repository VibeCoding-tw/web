光電感測器原理
與 方波訊號

看不見的數位之眼：從光的遮斷到精確的速度測量。

👁️ 1. 光電遮斷器：數位化的眼睛 (20 min)

物理構造：理解發光二極體（Emitter）與光敏電晶體（Receiver）的對稱配置。

遮斷原理：當編碼盤（Encoder Disk）轉動時，葉片會交替「阻擋」與「通過」光線。

訊號轉換：光線通過時輸出低電位（0V），遮斷時輸出高電位（3.3V/5V）。

🔍 學習任務

觀察家中的滾輪滑鼠或老式印表機，思考裡面哪裡運用了類似的結構？

編碼盤轉動示意圖

📈 2. 方波訊號 (Square Wave) 與頻率 (20 min)

  • 振幅 (Amplitude)：電壓的高低 (0V - 3.3V)。
  • 頻率 (Frequency)：每秒鐘產生多少個脈衝（Pulse）。
  • 佔空比 (Duty Cycle)：高電位在一個週期內所佔的時間比例。

🧮 轉速計算數學

已知編碼盤一圈有 $N$ 個齒，每秒測得 $P$ 個脈衝。
則每秒轉速 $RPS = P / N$

🎯 學習任務

計算若編碼盤有 20 個齒，測得頻率為 100Hz，這台車的馬達每分鐘轉多少圈 (RPM)？

數位方波示意圖 (0 與 1 的切換)

📉 3. 雜訊與遲滯 (Hysteresis) (15 min)

解釋紅外線發射與接收對管如何透過「柵格盤」產生高低電平（0 與 1）。

解析度 (PPR)：Pulses Per Revolution。例如：10 格的柵格盤轉一圈產生 10 個脈衝。

⚠️ 預習任務

思考當編碼盤停在「半遮半掩」的位置時，訊號會不會在 0 與 1 之間瘋狂跳動？這對計算會有什麼影響？

📏 學習目標

計算公式練習——已知輪胎直徑與 PPR，如何算出「一個脈衝代表前進了多少公厘」。

實作任務

物理量 -> 電訊號 -> 數據 -> 資訊的完整轉換鏈。

📊 60 min

【作業 1-1】訊號觀測站

任務內容：手動轉動馬達，觀察光電感測器的輸出波形。

  • 波形捕捉：確認方波邊緣銳利度
  • 轉速線性測試：觀察脈衝密度變化
  • 邊界分析：尋找臨界狀態的高頻雜訊
⚡ 60 min

【作業 1-2】外部中斷計數器

任務內容：在 ESP32 上實作 attachInterrupt() 精確計數。

  • 中斷設定：使用 RISING 觸發
  • 效能檢驗：驗證計數準確度
  • 去彈跳 (Debounce)：軟體濾除雜訊
⏱️ 45 min

【作業 1-3】即時轉速計

任務內容：將計數值轉化為網頁上的「時速儀表板」。

  • 公式代入：換算脈衝為 cm/s
  • 數據傳輸：HTTP/WebSocket 即時傳送
  • 校準實驗：前進 1 公尺誤差驗證

進階 11 - 單元 1 | 光電感測器原理與方波訊號

訊號取樣與基礎計數

【作業 1-1】訊號觀測站 (方波品質)

  • GitHub 角色： 訊號分析師。
  • 執行方式： 學生需提交「示波器截圖」或「Serial Plotter 截圖」。
  • 觀察點： 轉動馬達時，高低電位的切換是否乾淨？有無明顯的雜訊毛邊？
  • 評分重點： 能否分辨什麼是「數位訊號」的邊緣（Rising/Falling Edge）。

【作業 1-2】外部中斷計數器 (精確度)

  • GitHub 角色： 數據採集員。
  • 執行方式： 學生提交 interrupt_counter.cpp，能即時列印累計脈衝數。
  • 實驗： 手動轉動輪子一圈，檢查計數器是否剛好增加「編碼器格數」（如 20 格）。
  • 優勢： 這是建立對「中斷觸發」信任感的基礎。

【作業 1-3】即時轉速計 (數據轉換)

  • GitHub 角色： 儀表板開發者。
  • 執行方式： 將格數轉換為物理長度（cm）或轉速（RPM）。
  • 呈現： 透過 Web 介面顯示目前的「虛擬里程計」與「當前時速」。
  • 評分重點： 公式換算的正確性，特別是單位轉換（格 -> 圈 -> 公分）。
