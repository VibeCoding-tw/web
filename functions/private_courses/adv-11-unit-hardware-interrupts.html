🤔 1. 什麼是中斷？生活中的類比 (15 min)

🐢 輪詢 (Polling)

你正在煮泡麵，每隔 5 秒就跑去廚房看水開了沒。這讓你無法專心寫作業（浪費 CPU 資源）。

⚡ 中斷 (Interrupt)

你回房間寫作業，等水開了水壺發出「鳴笛聲」，你才放下筆去關火。這讓你效率極大化。

關鍵觀念：中斷能讓處理器從繁重的任務中被「強行暫停」，去處理更緊急的事。

📜 2. 中斷服務函式 (ISR) 的三大法則 (25 min)

當 CPU 被中斷後，會跳轉去執行一段特殊的程式碼，稱為 ISR (Interrupt Service Routine)。

🚀

短小精悍

ISR 必須極快地執行完畢，否則會阻塞主程式和其他中斷。

🚫

禁止延遲

在 ISR 內絕對不能使用 delay() 或複雜的 Serial.print()。

🔄

變數共享

在 ISR 中被修改的變數，必須宣告為 volatile（告訴編譯器：這個數值隨時會變，不要亂優化）。

🎚️ 3. 中斷觸發類型 (10 min)

  • RISING
    上升沿： 電位從 0 變 1 的瞬間。常用於按鈕按下瞬間。
  • FALLING
    下降沿： 電位從 1 變 0 的瞬間。常用於按鈕放開瞬間。
  • CHANGE
    改變時： 只要電位有動靜就觸發。常用於編碼器計數。

B. 參考範例代碼 (Reference Example Code)

src/basic_interrupt.cpp (中斷法框架)

volatile unsigned long last_time = 0;
volatile int count = 0;

void IRAM_ATTR handleInterrupt() {
  unsigned long now = millis();
  if (now - last_time > 50) { // 軟體去彈跳
    count++;
  }
  last_time = now;
}

void setup() {
  Serial.begin(115200);
  attachInterrupt(D2, handleInterrupt, FALLING);
}

void loop() {
  Serial.printf("Count: %d\n", count);
  delay(100);
}

src/polling_check.cpp (傳統輪詢法對照)

void loop() {
  if (digitalRead(D2) == LOW) {
    // 輪詢法容易因 loop 內的其他延遲（如以下 delay）而漏計
    count++; 
    delay(5); // 模擬去彈跳
  }
  delay(100); // 模擬其他繁重任務
}

README.md (分析報告樣板)

# 中斷效能分析報告
- 測試環境：ESP32-C3 + 光電編碼器
- 觀察結果：
  1. 輪詢法在全速轉動時丟失約 30% 脈衝。
  2. 中斷法透過 IRAM_ATTR 實現微秒級響應。
  3. 軟體去彈跳閾值設定為 50ms 最為穩定。

C. 自動評分 (Autograding) 的設計

  • 邏輯單元測試：： 模擬按鈕抖動，驗證 ISR 是否具備正確的軟體去彈跳機制。
