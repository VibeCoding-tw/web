🤔 1. 什麼是中斷？生活中的類比 (15 min)

🐢 輪詢 (Polling)

你正在煮泡麵，每隔 5 秒就跑去廚房看水開了沒。這讓你無法專心寫作業（浪費 CPU 資源）。

⚡ 中斷 (Interrupt)

你回房間寫作業，等水開了水壺發出「鳴笛聲」，你才放下筆去關火。這讓你效率極大化。

關鍵觀念：中斷能讓處理器從繁重的任務中被「強行暫停」，去處理更緊急的事。

📜 2. 中斷服務函式 (ISR) 的三大法則 (25 min)

當 CPU 被中斷後，會跳轉去執行一段特殊的程式碼，稱為 ISR (Interrupt Service Routine)。

🚀

短小精悍

ISR 必須極快地執行完畢，否則會阻塞主程式和其他中斷。

🚫

禁止延遲

在 ISR 內絕對不能使用 delay() 或複雜的 Serial.print()。

🔄

變數共享

在 ISR 中被修改的變數，必須宣告為 volatile（告訴編譯器：這個數值隨時會變，不要亂優化）。

🎚️ 3. 中斷觸發類型 (10 min)

  • RISING
    上升沿： 電位從 0 變 1 的瞬間。常用於按鈕按下瞬間。
  • FALLING
    下降沿： 電位從 1 變 0 的瞬間。常用於按鈕放開瞬間。
  • CHANGE
    改變時： 只要電位有動靜就觸發。常用於編碼器計數。

實作任務

從理論到實戰，掌握即時運算的核心。

⏱️ 60 min

【作業 2-1】光速反應測試

任務內容：比較 CPU 忙碌時，Polling 與 Interrupt 對按鈕的反應速度。

  • 實驗 A (Polling)：忙碌迴圈中穿插檢查。
  • 實驗 B (Interrupt)：attachInterrupt 隨按隨亮。
  • 觀測重點：體會「非同步且不可預知」事件的處理差異。
🦗 60 min

【作業 2-2】彈跳惡夢

任務內容：解決按一下按鈕卻觸發 20 次中斷的問題。

  • 問題再現：按鈕計數器亂跳。
  • 軟體去彈跳：ISR 內加入 50ms 時間差判斷。
  • 驗證：確保按一下就是加一，無論按多快。
🏎️ 60 min

【作業 2-3】馬達編碼器實戰

任務內容：利用光電感測器方波訊號觸發中斷，計算輪子轉動格數。

  • 精確計數：ISR 內累加脈衝數。
  • 負載測試：全速轉動馬達確認無漏計。
  • 目標：將觀念應用於高頻數據擷取（>1kHz）。

作業指引 (Assignment Guide)

作業 2-1：光速反應測試 (中斷 vs 輪詢)

  • 角色扮演： 對照組實驗室。
  • 執行方式： 比較 CPU 忙碌時，Polling 與 Interrupt 對按鈕的反應速度。
  • 評分重點： 確認學生配置 attachInterrupt 的觸發模式，並理解中斷搶佔 CPU 的原理。

作業 2-2：彈跳惡夢 (軟體去彈跳)

  • 角色扮演： 邏輯驗證中心。
  • 執行方式： 解決按一下按鈕卻觸發多次中斷的問題，在 ISR 內加入 50ms 時間差判斷。
  • 評分重點： 檢查變數是否宣告為 volatile，及時間判斷邏輯是否正確。

作業 2-3：馬達編碼器實戰 (高頻計數)

  • 角色扮演： 效能儀表板。
  • 執行方式： 利用光電感測器方波訊號觸發中斷，計算全速轉動時的脈衝數。
  • 評分重點： 確認 ISR 程式碼簡潔且無 delay() 或 Serial.print() 等漏計處理邏輯。
