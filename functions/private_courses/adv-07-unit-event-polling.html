進階 7 - 單元 4

效率之巔：
事件驅動 vs. 輪詢更新

數據獲取的藝術。學習如何在「極速反應」與「極致省電」之間取得完美平衡。

🍕 1. 觀念拆解：外送員與餐廳的溝通 (25 min)

  • 輪詢 (Polling)

    像是一個飢餓的顧客，每隔 1 分鐘就打電話問餐廳「我的飯好了嗎？」。

    ✅ 邏輯簡單、可預測執行時間 ❌ 浪費資源、高手延遲

  • 事件驅動 (Event-Driven)

    像是顧客點完餐後去做自己的事，等餐廳主動「傳簡訊」通知領餐。

    ✅ 省電、零閒置浪費、即時反應 ❌ 程式流程跳躍、初學者難追蹤

🔄 2. 瀏覽器中的事件與循環 (25 min)

// 事件驅動 vs 輪詢實作範例
// 事件驅動
btn.addEventListener('click', () => { ... });
// 輪詢更新
setInterval(() => {
if (checkStatus()) { handleResult(); }
}, 100);
  • addEventListener：理解瀏覽器如何監聽藍牙斷線、按鈕點擊。
  • setInterval/setTimeout：理解如何利用定時器實作輪詢。

🔍 學習任務

思考：為什麼「搖桿控制」適合輪詢，而「藍牙連線成功」適合事件驅動？

🚨 3. 硬體端的中斷 (Interrupts) (10 min)

ESP32 attachInterrupt：理解當網頁傳送「緊急停止」或車子撞到東西時，硬體如何立即跳出主程式進行處理（事件驅動的極致表現）。

🧠 進階思考

比較 WebSocket.onmessage (事件驅動) 與 setInterval + Fetch (輪詢) 的優缺點。何時該追求「極低延遲」，何時該追求「省電穩定」？

高手挑戰：極致通訊優化

從理論到實作，對決你的瀏覽器與硬體效能。

⏱️ 60 min

【作業 4-1】效率大挑戰

任務內容：優化 WiFi 密碼輸入介面的按鈕啟動邏輯。

  • 階段 A (輪詢)：使用 `setInterval` 檢查輸入框。
  • 階段 B (事件驅動)：改用 `input` 事件監聽器。
  • 觀測：使用 Performance Tab 觀察 CPU 消耗差異。
🕹️ 60 min

【作業 4-2】手感微調

任務內容：實作一個混合模式 (Hybrid) 的搖桿發送器。

  • 解決事件雨：在高頻 `mousemove` 下防止 BLE 崩潰。
  • 緩衝策略：將座標存入變數，但定時發出。
  • 目標：兼顧即時性與通訊頻寬穩定。
🆘 45 min

【作業 4-3】緊急停止

任務內容：模擬事件驅動在「安全性」上的應用。

  • 斷線監聽：使用 `gattserverdisconnected` 事件。
  • 立即反應：觸發時跳出警告並重置狀態。
  • 延遲對比：記錄「定時詢問」與「事件監聽」的反應秒數差異。
