🔄 1. 有限狀態機 (FSM) (20 min)

無人車不是「隨機」運行的，它在不同時刻處於不同「狀態」。 FSM 用來管理這些狀態的切換。

⚡ 核心概念

  • 狀態 (States)：系統當下的模式（如：搜尋線條、循跡中、遇到障礙物停止）。
  • 觸發條件 (Triggers)：從狀態 A 跳轉到狀態 B 的「關鍵事件」（如：感測器讀值全白時，進入搜尋狀態）。

♾️ 2. 控制迴圈 (The Main Loop) (20 min)

程式的核心是「永不停歇的思考」。輸入 -> 處理 -> 輸出。

📥 1. 感測 (Input)

讀取影像質心、檢查藍牙指令。

🧠 2. 決策 (Process)

計算 PID、判斷當前狀態。

📤 3. 執行 (Output)

更新 PWM 到馬達、更新網頁數據。

🚫 阻塞 (Blocking) 的威脅

理解為什麼在迴圈內使用 delay() 會導致車子失控。

為什麼？

因為延遲期間，CPU 是「睡著」的，感測器是盲目的。 車子若正在全速衝刺，會無法即時煞車。

學習任務：思考如何不用 delay() 也能讓 LED 每隔一秒閃爍一次？（提示：millis()）

作業指引 (Assignment Guide)

作業 3-1：「偽代碼建築師」：邏輯規劃

  • 角色扮演： 架構設計師。
  • 執行方式： 在不寫實際 C++ 代碼的情況下，用中文偽代碼描述整個 PID 迴圈的執行流程。
  • 評分重點： 能否清晰標註出「讀取」、「計算」、「輸出」與「採樣控制」的先後順序及邏輯判斷。

作業 3-2：「採樣頻率大探索」：時間穩定性

  • 角色扮演：： 性能測試員。
  • 執行方式： 故意將採樣時間從 10ms 調整到 100ms，觀察無人車循航穩定性的急遽惡化。
  • 評分重點： 紀錄不同採樣頻率下的控制波形，並推導出適合本硬體設備的「黃金採樣週期」。

作業 3-3：「非阻塞代碼重構」：millis() 實戰

  • 角色扮演： 代碼優化師。
  • 執行方式：： 將一段包含 delay() 的舊代碼改寫為基於 millis() 的狀態機結構。
  • 評分重點： 代碼執行時能否同時閃爍 LED 燈而不影響 PID 的精準計時（多工驗證）。
