⚡ 為什麼選擇 ESP32？ (15 min)

相較於傳統 Arduino Uno，ESP32 是一個質的飛躍。

🧠

雙核心 (Dual-Core)

擁有高達 240MHz 的運算速度。兩個核心分工合作：一個處理繁重的無線通訊，另一個專注執行你的程式邏輯。

📡

無線連線能力

內建 Wi-Fi 與 雙模藍牙 (經典藍牙 + BLE)。這是它成為 IoT 神器的關鍵，讓你的裝置能直接連上雲端。

🔌

豐富的外設

內建觸控感應、霍爾傳感器、硬體加密加速器，以及多種低功耗模式，適合電池供電的應用場景。

ESP-WROOM-32 模組

那塊金屬屏蔽罩下藏著真正的 CPU 與閃存 (Flash)，保護晶片不受外界訊號干擾。

2

USB 轉串口晶片 (CP2102/CH340)

電腦與 ESP32 溝通的翻譯官。如果燒錄失敗，通常是需要安裝這個晶片的驅動程式。

3

電壓調節器 (LDO)

ESP32 工作電壓是 3.3V。LDO 負責將 USB 的 5V 降壓。千萬別把 5V 接到 3.3V 腳位，會燒毀晶片！

4

EN 與 BOOT 按鈕

EN (Reset): 重啟開發板。
BOOT: 按住不放再按 EN，可以強制進入「燒錄模式」。

💻 軟體環境預熱 (25 min)

為了讓電腦聽懂 ESP32 的語言，我們需要準備好開發環境與驅動程式。

1. 安裝 IDE

推薦使用 VS Code + PlatformIO，這是專業開發者的首選。但 Arduino IDE 也是不錯的入門選擇。

2. 安裝驅動程式

確認電腦能識別 COM Port。
🔍 搜尋關鍵字: "CP2102 Driver" 或 "CH340 Driver" (依板子型號而定)

⚠️ 新手注意事項

  • ⚡ Micro USB 線材： 很多線只能充電不能傳輸資料。如果不亮燈或電腦沒反應，請先換一條線！
  • 🔌 正確插拔： 拿住接頭而不是拉扯線身，保護脆弱的 USB 接口。

作業指引 (Assignment Guide)

1. GitHub Classroom 設定

  • 點擊儀表板「領取作業」按鈕，獲取 `basic-01-esp32-arch` 作業儲存庫。
  • 將專案 Clone 至本地，並在 PlatformIO 中開啟。
  • 檢視 `src/main.cpp` 中預設的硬體資訊查詢代碼。

2. 翻轉課堂學習策略

  • 課前預習：觀看「ESP32 硬體架構」教學影片，認識核心組成。
  • 課中實作：進行硬體偵錯實驗，觀察不同狀態下的電壓變化。
  • 課後思考：為什麼 ESP32 需要設計兩個核心？在什麼場景下會有優勢？

3. 單元實作任務

  • 作業 1-1 (40%)：辨識開發板上的主要晶片型號（如 CP2102, AMS1117）並完成填空。
  • 作業 1-2 (30%)：撰寫代碼控制板載 LED 以 1Hz 頻率閃爍。
  • 作業 1-3 (30%)：透過 Serial 印出 ESP32 的 Chip Model 與 Core Count。

GitHub Classroom 詳細操作指南

第一步：準備範本倉庫 (Template Repo)

  1. 上傳包含基礎 Serial 設定但「未實作晶片資訊查詢」的 Repo 框架，並勾選 "Template repository"。

第二步：建立 Classroom 作業

  1. 建立作業並連結範本。建議開啟 Feedback Pull Requests 選項以便對比學生的硬體識讀分析與系統資源調配邏輯。

第三步：學生端操作 (Flipped Workflow)

第四步：教師回饋 (Review)

  1. 打開 Feedback PR，針對學生的「硬體架構圖解」與「電源穩定性分析（Brownout 預防）」予以具體回饋。

B. 參考範例代碼 (Reference Example Code)

main.cpp (讀取系統資訊)

#include <Arduino.h>
#include "esp_system.h"
#include "esp_spi_flash.h"

void setup() {
  Serial.begin(115200);
  delay(1000);

  // 1. 取得晶片資訊
  esp_chip_info_t chip_info;
  esp_chip_info(&chip_info);

  Serial.println("--- System Info ---");
  Serial.printf("CPU Cores: %d\n", chip_info.cores);
  Serial.printf("Flash Size: %d MB\n", spi_flash_get_chip_size() / (1024 * 1024));
  Serial.printf("Rev: %d\n", chip_info.revision);
}

void loop() {}

C. 自動評分 (Autograding) 的設計

  • 語法掃描： 檢查 main.cpp 是否包含了 esp_chip_info 相關函數調用。
  • 邏輯健壯性： 驗證學生在 README 中對「3.3V 穩壓器」位置的標註是否與目前實機使用的開發板一致。
