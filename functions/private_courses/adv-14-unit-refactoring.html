進階 14 - 單元 3

代碼重構與模組化設計

從「義大利麵代碼」進化到優雅的軟體架構。學習如何償還技術債，設計高內聚、低耦合的系統。

為什麼要重構？

為了趕進度而寫的快餐代碼是「技術債 (Technical Debt)」，利息就是未來修改時的高昂代價。重構不是增加新功能，而是讓代碼「更健康、更易讀、更好改」。

模組化原則

  • ✓高內聚 (High Cohesion)：一個檔案只做一件事
  • ✓低耦合 (Low Coupling)：模組間依賴越少越好
  • ✓關注點分離 (SoC)：感知、決策、執行分家

1 識別「代碼壞味道」 (Code Smells)

像聞牛奶一樣，有些代碼一打開就有「味道」。學會識別這些徵兆，是重構的第一步。

🐢 冗長函式 (Long Method)

一個 loop() 裡面寫了 500 行，包含連線、影像處理、馬達控制。看都看不完，更別說改了。

🔁 重複代碼 (Duplication)

違反 DRY (Don't Repeat Yourself) 原則。同樣的 JSON 解析邏輯複製貼上在三個地方，改了一個忘了改另外兩個。

🧱 硬編碼 (Hardcoding)

IP 地址、PID 參數直接寫死在程式碼深處。每次要微調參數都要重新編譯燒錄，浪費生命。

2 系統分層架構 (Separation of Concerns)

將你的無人車拆解為三個獨立的大腦區域，就像神經系統一樣。

👁️ 感知層 (Perception)

負責「看」與「聽」。影像讀取、感測器濾波、特徵提取。

Camera, IMU, LiDAR

🧠 決策層 (Logic)

負責「想」。狀態機切換 (FSM)、PID 運算、路徑規劃。

State Machine, PID

⚡ 執行層 (Action)

負責「做」。馬達 PWM 輸出、LED 顯示、蜂鳴器。

Motor, LED, Buzzer

📝 實作任務 (Code Challenges)

Task 3-1 「手術台」：解構大雜燴程式碼 (60 min)

任務說明

教師將提供一段「所有功能都塞在 loop() 裡」的義大利麵程式碼。你需要扮演外科醫生，將其拆解為乾淨的函式。

  • 提取函式 (Extract Function)：將影像處理、誤差計算、BLE 發送封裝成獨立的 Function。
  • 參數物件化：將散落的全局變數（Kp, Ki, Speed）封裝進 Config 物件。
  • 重命名演習：將 a, b1, temp 改為具語義的 errorCentroid, isMotorEnabled。

Task 3-2 打造「通訊模組」Class (60 min)

任務說明 (OOP)

主程式不應該知道底層是 Web Bluetooth API 還是 WebSocket。請利用 JavaScript Class 封裝通訊邏輯。

  • 建立 BLEManager：實作 connect(), sendJson(), onNotify() 方法。
  • 介面隔離：確保主程式只呼叫 ble.send({cmd: 'run'})，而不直接操作 characteristic。
  • 多實例思考：如果未來有兩台車，你的 Class 能直接 new BLEManager() 兩次嗎？

Task 3-3 「插件化」影像處理鏈 (45 min)

任務說明 (Design Pattern)

設計一個可以自由組合濾鏡的「處理鏈 (Pipeline)」。目標是「對擴展開放，對修改封閉」。

  • 處理鏈陣列：建立 processors = [Grayscale, Threshold, Centroid]。
  • 動態擴充：嘗試在不修改主迴圈的情況下，直接 push 一個「中值濾波」模組到陣列中並生效。
🧩

重構的藝術

好的代碼應該像一篇優美的散文，讓人讀起來賞心悅目。當你能夠寫出別人願意維護的代碼時，你就正式晉升為資深工程師了。
