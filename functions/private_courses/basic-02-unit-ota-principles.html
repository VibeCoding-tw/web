🔄 OTA 的核心：雙分區機制 (20 min)

如何確保更新過程中斷電，機器也不會壞掉？

1

預防「磚頭化」 (Bricking)

絕不能直接覆蓋正在執行的程式！OTA 採用「先下載到備份區，驗證成功後再切換」的策略。

2

分區切換 (A/B Partition)

ESP32 將 Flash 分為 app0 與 app1。當 app0 正在執行時，OTA 會將新韌體寫入 app1。反之亦然。

3

驗證與啟動標記

寫入完成後會檢查 Checksum。成功後修改 `otadata` 分區，指引下次開機從新分區啟動。

🌐 Web OTA 的傳輸流程 (25 min)

1. HTTP POST

Web OTA 本質上是將 .bin 檔案透過瀏覽器的 `FormData` 上傳到 ESP32 的特定路由。

2. 串流寫入

ESP32 收到數據時，直接調用 `Update.write()` 將二進制流寫入 Flash 的特定地址。

3. 進度回報

透過 JavaScript 的 `XMLHttpRequest` 監聽 `progress` 事件，實現視覺化進度條。

browser-upload.js

前端進度監聽範例

xhr.upload.addEventListener('progress', function(e) {
  if (e.lengthComputable) {
    const percent = Math.round((e.loaded / e.total) * 100);
    progressBar.style.width = percent + '%';
    statusText.innerText = `上傳中: ${percent}%`;
  }
});

🔐 安全與驗證 (15 min)

如果隨便一個連上 Wi-Fi 的人都能上傳韌體，會發生什麼事？安全機制是生產等級產品的必修課。

  • ✔ 身份驗證： 在更新頁面前增加密碼彈窗或 Token 驗證。
  • ✔ mDNS (網路發現)： 讓設備在網路上以 `esp32.local` 身份出現。
  • ✔ ElegantOTA： 解析此庫如何封裝 驗證 -> 寫入 -> 切換 的過程。

B. 參考範例代碼 (Reference Example Code)

main.cpp (基礎 ArduinoOTA 設置)

#include <WiFi.h>
#include <ArduinoOTA.h>

void setup() {
  Serial.begin(115200);
  WiFi.begin("SSID", "PASS");
  
  // 設置 OTA 屬性
  ArduinoOTA.setHostname("esp32-updater");
  
  ArduinoOTA.onStart([]() {
    Serial.println("Start updating...");
  });
  ArduinoOTA.onEnd([]() {
    Serial.println("\nEnd");
  });
  ArduinoOTA.onProgress([](unsigned int progress, unsigned int total) {
    Serial.printf("Progress: %u%%\r", (progress / (total / 100)));
  });
  
  ArduinoOTA.begin();
}

void loop() {
  ArduinoOTA.handle();
}
